Wedson Almeida Filho (Old Google account) (Aug 03 2021 at 12:56):

Generally, we don't want to expose C data structures to non-kernel-crate.

The approach we've taken in the existing code, and gpio/irq/power/amba/pci code soon to come is similar to File, FileRef and Task where we wrap raw pointers and only expose functionality currently needed.

Part of the reason we don't want to expose raw pointer to C data is because it requires unsafe code to access, it has no information about lifetime (and for how long callee can hold on to pointers), it lacks information about which locks need to be acquired to access which fields, and there's a mismatch wrt memory model. IOW, exposing C data structure to Rust is a minefield that we think is better not exposed for now.
