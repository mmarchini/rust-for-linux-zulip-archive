Hi :-) I didn't want to put this on the mailing lists because maybe it's a stupid idea and it's not something I'm able to implement myself. There was some talk about how to best utilize Rust's benefits. I was just wondering if maybe filesystems wouldn't be a good fit for a real driver -- they're "leaf" drivers, but interact very directly with users; they are independent of hardware; and correctness and safety guarantees is something that everybody wants from their filesystem. Maybe it would even be possible to write the code in such a way that you could compile it for userspace too and run unit tests against it without having to build and run a full kernel? (Which would make it infinitely more hackable than any current kernel filesystem implementation...)
