Topic: `addr_of` is not necessary for `Task::pid`?

Boqun Feng (Dec 12 2023 at 03:21):

Currently we have the following impl:

impl Task {
     pub fn pid(&self) -> Pid {
          // SAFETY: By the type invariant, we know that `self.0` is a valid task. Valid tasks always
          // have a valid pid.
          unsafe { *ptr::addr_of!((*self.0.get()).pid) }
     }
}

However, I wonder whether

unsafe { (*self.0.get()).pid }

just works?

Since here we don't create reference to pid, and (*self.0.get()) won't create temporary references to the task_struct, so the above should work without any problem. @Wedson Almeida Filho
