Topic: Box<FnOnce()> is special

Boqun Feng (Oct 27 2024 at 16:55):

@Danilo Krummrich , I hit a problem where it seems Box<FnOnce()> is different than BoxLike<FnOnce()>. Basically, you can call a Box<FnOnce()> with f() but you cannot do it with BoxLike:

// this won't compile.
fn call_once<F: FnOnce()>(f: BoxLike<F>) {
    f()
}

We can move the value out of box, but that means an extra move involved here:

fn call_once_move<F: FnOnce()>(f: BoxLike<F>) {
    let f = Box::into_inner(f.0);
    f()
}
