Martin Rodriguez Reboredo (Jul 16 2022 at 00:46):

When using Clang 14 with Bindgen (at least in Arch Linux) did you noticed that the write field from file_operations is using a mut* instead of a const*? Currently I have

// snip
#[repr(C)]
#[derive(Copy, Clone)]
pub struct file_operations {
    pub write: ::core::option::Option<
        unsafe extern "C" fn(
            arg1: *mut file,
            arg2: *mut c_types::c_char,
            arg3: usize,
            arg4: *mut loff_t,
        ) -> isize,
    >,
// more code

But instead we need

    pub write: ::core::option::Option<
        unsafe extern "C" fn(
            arg1: *mut file,
            arg2: *const c_types::c_char,
            arg3: usize,
            arg4: *mut loff_t,
        ) -> isize,
    >

And to make this work I needed to do some sed sorcery to change it into the same thing

--- a/rust/Makefile
+++ b/rust/Makefile
@@ -304,6 +304,8 @@ quiet_cmd_bindgen = BINDGEN $@

 $(obj)/bindings_generated.rs: private bindgen_target_flags = \
     $(shell grep -v '^\#\|^$$' $(srctree)/$(src)/bindgen_parameters)
+$(obj)/bindings_generated.rs: private bindgen_target_extra = ; \
+    sed -Ei '/struct file_operations/,/arg2: \*/ s|(arg2: \*.*)|\1\n//m|' $@; sed -Ei '/^\/\/m/,/arg2/ s|(arg2: \*)mut|\1const|' $@
 $(obj)/bindings_generated.rs: $(src)/kernel/bindings_helper.h \
     $(src)/bindgen_parameters FORCE
    $(call if_changed_dep,bindgen)

Did this happen to someone?

Martin Rodriguez Reboredo (Jul 23 2022 at 17:28):

Opened up Rust-for-Linux/linux#835 for reviews. Happens that when I was doing a BTF with pahole build it expanded a definition, namely BTF_TYPE_TAG(value), to be __attribute__((btf_type_tag(#value))), this confounds bindgen and makes it discard the const qualifier in attributed function parameters. That's why write got a mut* in the bindings.
