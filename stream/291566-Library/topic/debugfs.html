Hi,

I decided to get started with Rust in the kernel. I consider myself to be a novice++ in Rust, and I only used it a few times to write a few CLI tools for myself, but I'm now a big fan of it.

I wanted to start with something pretty small, and work my way up from there, that's why I chose to write an abstraction for the debugfs API. I didn't find any reference of debugfs abstraction on zulip nor on github, so hopefully no one else is working on that.
You can find my dev branch over here: https://git.codelinaro.org/fabien.parent/linux/-/commits/rust-debugfs (sample here: https://git.codelinaro.org/fabien.parent/linux/-/commit/b1ce93e4f7c172d1b3f86e1ddd4684cdc8af6dbf#61f1b30b6476bc607fa5048282466286c564b802). This is still work in progress, it is functional, but is not ready for reviewing yet.
The branch is based on rust-next and depends on the File::Operations abstraction which is not upstream yet.

I'm also writing a driver for a QCOM IP, that will use this abstraction. I will share that driver at a later time.
