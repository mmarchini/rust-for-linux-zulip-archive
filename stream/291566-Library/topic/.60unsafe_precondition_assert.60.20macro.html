Ritvik Gupta (Jul 12 2025 at 13:09):

Hello!
I was working on adding support for unsafe_precondition_assert and had couple of questions:

1: This is the macro usage of core::assert_unsafe_precondition [nightly-experimental] from rust/library/core/src/str/traits.rs:

#[inline]
#[track_caller]
unsafe fn get_unchecked_mut(self, slice: *mut str) -> *mut Self::Output {
    let slice = slice as *mut [u8];

    assert_unsafe_precondition!(
        check_library_ub,
        "str::get_unchecked_mut requires that the range is within the string slice",
        (
            start: usize = self.start,
            end: usize = self.end,
            len: usize = slice.len()
        ) => end >= start && end <= len,
    );

    // SAFETY: see comments for `get_unchecked`.
    unsafe {
        let new_len = unchecked_sub(self.end, self.start);
        ptr::slice_from_raw_parts_mut(slice.as_mut_ptr().add(self.start), new_len) as *mut str
    }
}

For the kernel side, I am considering two possible macro signatures:

  1. Multiple pre-conditions (for grouping multiple conditions under single message):
unsafe_precondition_assert!(
    "Pointer must be valid. Length and content must match",
    [
        (!ptr.is_null()),
        (ptr.is_aligned()),
        (string.len() != slice.len()),
        (string == String::from_utf8_lossy(slice)),
    ]
);
  1. Single pre-condition (repeat multiple times for multiple checks or combine with &&):
unsafe_precondition_assert!("Ptr must be valid", !ptr.is_null() && ptr.is_aligned());
unsafe_precondition_assert!("Length must be equal", string.len() == slice.len());
unsafe_precondition_assert!("Content must be same", string == String::from_utf8_lossy(slice));

Support both or something else? How'd you like to write them?

2: Should this be placed in

  • A new file (ex: kernel/ub_checks.rs, similar to how it's structured in core),
  • Existing kernel/lib.rs or something else?

Open to suggestions :)
Thanks!
