bjorn3 (Nov 26 2022 at 22:16):

I think your proposal is actually more complicated than quote. It needs a proc macro to be implemented while quote is a decl macro. In addition something like get_${attributes[0].name} being a single identifier is not implemented at all as it parsed the same as get_ ${attributes[0].name} which would be two identifiers. As such proc macros can't distinguish between the two.

bjorn3 (Nov 26 2022 at 22:20):

Quote's syntax is really simple. It only has #var for taking a thing stored in the variable var and converting it into a token tree before pasting it and it had #()* for repetition. Both mirror the decl macro syntax, except replacing $ with #.

bjorn3 (Nov 26 2022 at 22:21):

Sure, quote's actual implementation is conplicated, but that is necessary to prevent exponential runtime in terms of input size. (rustc's decl macro expansion has some cases where it runs in exponential time. quote avoids these cases.)
