Trevor Gross (Feb 09 2024 at 23:02):

[u8] is unsized, Box<[u8]> is sized as two pointers (start + len), aka "fat pointer" - so pointer plus metadata. A Box<dyn SomeTrait> is also two pointers, as is &[u8] and &dyn SomeTrait.

ThinBox has the same information as Box except that metadata is stored at the beginning of the allocation instead of wherever the Box is. Benefits: now you only have one pointer to pass around, which makes it easier to work with some APIs that give you a single pointer for object storage (Wedson's use case). Downsides: knowing the length now means you need to spend time dereferencing another value
