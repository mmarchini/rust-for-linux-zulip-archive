Found a workaround

// need to public `ArcInner` first
pub struct ArcInner<T: ?Sized> { ... }

// Introduce an external layer, may have users in other place as well
#[derive(Copy, Clone)]
pub struct Ptr<P: Deref>(P)

put type ArcBorrow<'a, T> = Ptr<&'a ArcInner<T>>;

We need Ptr since we want to:

impl Deref for ArcBorrow<'_, T>

, otherwise we can just use tuple type:

put type ArcBorrow<'a, T> = (&'a ArcInner<T>);

Wedson Almeida Filho (Sep 21 2023 at 14:27):

@baul I wrote ArcBorrow (then called RefBorrow) before the GAT feature was even complete. So the ForeignOwnable::Borrowed (then called PointerWrapper::Borrowed) associated type couldn't have a lifetime associated with it. So we had ArcBorrow for Arc (then called Ref) and UnsafeReference for Box.

When GATs were completed (and even before they were stabilised), we started using them. The commit below is available in the rust branch:

commit 40839c04dcd2d772367de3cac1e987271e78b5b6
Author: Wedson Almeida Filho <wedsonaf@google.com>
Date:   Fri Nov 12 21:22:09 2021 +0000

    rust: use generic associated types (GATs) to implement `PointerWrapper`.

    The feature, although still unstable, is no longer incomplete. It
    allows us to use a lifetime in the `Borrow` assocciated type, which
    allows us to simplify some of the code around borrowing the wrapped
    value and also allows us to wrap raw pointers and integers without
    violating `Deref` recommendations.

    Signed-off-by: Wedson Almeida Filho <wedsonaf@google.com>

At the time I removed UnsafeReference<T> in favour of &T but I didn't realise that I could do something similar for ArcBorrow. Thankfully @Boqun Feng realised this a few months back.

Well, better late than never! Here we are :)

Does it make sense to you now?
