Gary Guo (Mar 27 2026 at 15:58):

Currently I have something like this:

mb(Full);
mb(Acquire); // Map to mb() but documented to only need LOAD->{LOAD,STORE} ordering
mb(Release); // Map to mb() but documented to only need {LOAD,STORE}->STORE ordering
mb(Read); // Map to rmb
mb(Write); // Map to wmb

I have also had a design where the primitive is a pair (Before, After), so mb(Read) is short-hand for mb((Read, Read)) and mb(Acquire) is short-hand for mb((Read, Full)). This maps very well to hardware primitives, although it looks like this cannot represent AcqRel ordering (because it provides LOAD->LOAD, LOAD->STORE, STORE->STORE, so it cannot be represented as pair). So probably not worth having the complexity.
