Topic: improving build times

bjorn3 (Jul 09 2022 at 14:44):

I have been looking a bit into making kernel.o faster to compile. Yesterday I already found a way to significantly improve the performance of rust-analyzer by omitting testing code. Today I have been looking into splitting the bindgen generated bindings form the main kernel crate. This surprisingly resulted into a split of 10s for the bindings and 2s for the kernel crate. While the total compile time seems to be identical, this should improve incremental performance when changing the kernel crate but not any C header files. It may reduce performance for calling said default implementations a tiny bit though as they are no longer inlinable without LTO. Upon looking into why the bindings take so long too compile, I saw that pretty much the entire compiled bindings crate consists of Default implementations. While part of them use mem::zeroed(), the others manually call Default::default() for every field.
