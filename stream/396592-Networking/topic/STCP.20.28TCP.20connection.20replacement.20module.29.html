Lauri Jakku (Dec 21 2025 at 08:42):

bjorn3 kirjoitti:

If you allow any certificate, then it would be just as insecure as unencrypted connections against an active attacker. They can just present their own certificate. Also how does this compare against something like wireguard? That doesn't require any changes to individual applications at all, just needs an admin to configure a network interface and then tunels any network protocol for the specified ip address ranges.

Nope, user does NOT to have any knowledge of encryption details: Everything is handled dynamically. The module uses random private & publickeys and uses the kernel crypto for generating shared key. There is nothing that user have to do except change the socket's protocol number from IPPROTO_TCP => IPPROTO_STCP.

Everything is made dynamic, no need to know nothing from encryption, that is the beauty of SCP module.

Documentation of the handshake & everything else: https://github.com/MiesSuomesta/STCP/tree/main/kernel/OOT/linux/docs

The protocol can replace every TCP connection making them very secure with minimal effort.
