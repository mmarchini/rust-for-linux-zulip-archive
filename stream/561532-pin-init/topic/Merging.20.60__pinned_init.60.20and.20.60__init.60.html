Topic: Merging `__pinned_init` and `__init`

Gary Guo (Jul 22 2026 at 13:43):

Currently we have both __init and __pinned_init and the safety requirement of Init requires them to both do the same thing.

I wonder if we should just merge them to a single __init method, and document as safety requiremnt of Init that the pinning guarantee is cancelled out.

This should save some code duplication (and save some symbol name length by removing pinned_ from the name :slight_smile:).

Cc @Benno Lossin
