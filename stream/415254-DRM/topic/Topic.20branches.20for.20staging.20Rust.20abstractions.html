We got the first shot of topic branches for Rust abstractions in place.

We have the following branches:

  • nova/nova
  • nova/rust-device
  • nova/rust-common
  • linux-rust-pci/rust-pci
  • drm-misc/topic/rust-drm

Both, rust-pci and topic/rust-drm, are based on rust-device, which simply contains a few generic device abstractions.

There is also nova/rust-common, which basically got rust-device, rust-pci and rust-drm merged and acts as the integration branch we talked about. nova/nova contains the nova driver stub, based on rust-common, making use of the PCI and DRM abstractions.
