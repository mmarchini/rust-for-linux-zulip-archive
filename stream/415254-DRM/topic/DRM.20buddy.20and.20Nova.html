Topic: DRM buddy and Nova

Joel Fernandes (Nov 18 2025 at 21:13):

I just realized thanks to @John Hubbard that since CONFIG_DRM_BUDDY depends on CONFIG_DRM, and since nova-core has a hard dependency on not depending on CONFIG_DRM (since it can be used for other usecases like vGPU) ; we cannot use DRM_BUDDY as it is for the nova-core driver.

This leaves us with a couple of options:

  1. Refactor DRM_BUDDY so that the buddy algorithm it uses is extracted into external code(say in /lib/ or something along those lines).
    or,

  2. Implement the buddy allocator in Rust (say in rust/mm/).

The drawback of #2 is fragmentation and duplication.
The drawback of #1 is the logging dependencies of DRM_BUDDY on DRM devices will still live in drm/ so DRM drivers still need to call into DRM_BUDDY though the actual buddy allocator will be in common C code.

What do people think we should do? If Rust was supported by all architectures supporting DRM, then we could even make the common code as Rust code, but unfortunately we're not there yet.

CC @Dave Airlie @Danilo Krummrich @Alice Ryhl (input from everyone welcomed).
