Miguel Ojeda (Jan 22 2026 at 01:11):

Gary Guo said:

I would be good if we can also find cases where Clippy codegen differently w.r.t. rustc and report a bug upstream, too.

Clippy intentionally documents that they may not generate the same code, so certain things may not be a bug -- I sent this PR to document it after asking the mantainers a few years ago:

https://github.com/rust-lang/rust-clippy/pull/8037
https://github.com/rust-lang/rust-clippy/issues/8035

In particular, @bjorn3 explained back then this snippet I asked about indeed intentionally disables MIR optimizations which in turn also affect LLVM optimizations too:

 // FIXME: #4825; This is required, because Clippy lints that are based on MIR have to be
 // run on the unoptimized MIR. On the other hand this results in some false negatives. If
 // MIR passes can be enabled / disabled separately, we should figure out, what passes to
 // use for Clippy.
 config.opts.debugging_opts.mir_opt_level = Some(0);
