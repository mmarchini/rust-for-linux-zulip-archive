SeungJong Ha (Feb 06 2026 at 12:35):

Hi Andreas,

I've finished cleaning up the rebase. Here is a summary of the two branches:

1. rnull-v6.19-rc8

  • Base: rust-fixes (v6.19-rc8)

  • branch: https://github.com/Engineer-HaMa/linux/tree/rnull-v6.19-rc8

  • Ownable/Owned v14: Updated queue_rq signature to use Owned<IdleRequest<T>> and the new request lifecycle (IdleRequest → Request).

  • DMA API: Adapted to CoherentAllocation::alloc_coherent (removed generic type param, now takes &Device<Bound>).
  • TagSet: TagSet::new now takes 6 args (added flags: Flags), and GenDiskBuilder::build returns Result<Arc<GenDisk<T>>>.
  • Module macro: Updated authors to take a plural array (["Name"]).
  • Block ops: Adapted to current BlkResult, new_request_data(), etc.

2. rnvme-v6.19-rc8

  • Base: rnull-v6.19-rc8
  • branch: https://github.com/Engineer-HaMa/linux/tree/rnvme-v6.19-rc8

This branch adds 13 commits on top of rnull-v6.19-rc8, ported from your rnvme-v6.15 branch. Key changes beyond a straight rebase:

  • IRQ vectors: Changed to allocate all vectors upfront in probe() instead of re-allocating in setup_io_queues(). This fixes an issue where pci_alloc_irq_vectors fails to re-allocate when MSI-X is already enabled.
  • map_queues fix: Changed ForeignOwnable::from_foreign() (which takes ownership) to ForeignOwnable::borrow() for driver_data access. This prevents a use-after-free of the tag set data.
  • TagSetDataHandle<T>: Introduced a wrapper type to pass tag-set data to Operations::new_request_data() safely. This avoids exposing raw *mut c_void to drivers while working around Rust 1.83's lack of use<> precise capturing in trait methods (needed because ForeignBorrowed<'_, T> with impl PinInit return causes E0195).
  • PCI IRQ helpers: Added irq_by_index() / request_irq_by_index() for indexed IRQ vector registration.
  • Request queue refactoring: Split RequestQueue into RequestQueueRef (borrowed) and RequestQueue (owning), and moved SyncRequest to request_queue.rs.

Both branches were ported with the help of AI tools (Claude) and verified in a QEMU VM (virtme-ng) with a virtual NVMe device. I verified rnull insmod/rmmod and rnvme basic I/O (ext4 mount/read/umount).
I have applied some final adjustments using git commit --amend to polish the implementation. While this summary covers the key architectural changes, please treat the code in the repository as the source of truth if there are any slight discrepancies.

I hope this saves you some time when you get back to rnvme!
