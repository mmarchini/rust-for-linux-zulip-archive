Topic: `make rust-analyzer` alone is not enough on fresh builds

Onur Özkan (May 11 2026 at 08:47):

On a fresh checkout, running make rust-analyzer is not fully functional (e.g. broken proc macros, missing binding files). It currently requires either running a full build first or running prepare target before rust-analyzer. This is quite bad development experience because the issue only becomes apparent later when the LSP fails on a proc macro or binding types/functions.

I think we should either update the documentation to recommend make prepare rust-analyzer, or add prepare as a dependency of the rust-analyzer target by default. What do you think?
