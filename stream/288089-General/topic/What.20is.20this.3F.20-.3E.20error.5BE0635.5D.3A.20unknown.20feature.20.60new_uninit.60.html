Miguel Ojeda (Mar 21 2024 at 17:44):

new_uninit is an alloc feature, and the compiler complains about features that are "introduced" by alloc if it does not see alloc. I have seen this happen when upgrading alloc.

So e.g. use alloc; (or even extern crate alloc;) is enough to make it go away. Same if you use the kernel prelude, since it turn it references alloc.

Miguel Ojeda (Mar 21 2024 at 18:15):

One workaround would be to force alloc:

diff --git a/scripts/Makefile.build b/scripts/Makefile.build
index 0fb7a785594c..aa3a30a23762 100644
--- a/scripts/Makefile.build
+++ b/scripts/Makefile.build
@@ -272,7 +272,7 @@ rust_common_cmd = \
        -Zallow-features=$(rust_allowed_features) \
        -Zcrate-attr=no_std \
        -Zcrate-attr='feature($(rust_allowed_features))' \
-       --extern alloc --extern kernel \
+       -Zunstable-options --extern force:alloc --extern kernel \
        --crate-type rlib -L $(objtree)/rust/ \
        --crate-name $(basename $(notdir $@)) \
        --sysroot=/dev/null \
