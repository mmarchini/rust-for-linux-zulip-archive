Alice Ryhl (Apr 11 2024 at 10:25):

You might do something like this:

use core::ffi::c_void;

extern "C" fn run_callback<F: FnOnce()>(data: *mut c_void) {
    let data = data as *mut F;
    let data = unsafe { Box::from_raw(data) };
    (data)();
}

fn register_callback<F: FnOnce()>(func: F) {
    let boxed_func = Box::new(func);
    let data = Box::into_raw(boxed_func) as *mut c_void;
    unsafe { bindings::register_callback(data, run_callback::<F>) };
}

Wedson Almeida Filho (Apr 11 2024 at 14:48):

The example that Alice provided is one where your function is really a closure that carries data with it. Rust also allows you to pass function pointer like C, if that's what you want. In such cases, there is no memory allocation involved. Here's an example:

extern "C" {
    // Some C function that you want to call that takes a function pointer as argument.
    fn external_function_call(func: Option<extern "C" fn(*mut core::ffi::c_void) -> u32>);
}

// The callback implemented in Rust that C will call.
extern "C" fn my_rust_callback(_context: *mut core::ffi::c_void) -> u32 {
    0
}

fn main() {
    // Your call to the external function C function with a Rust callback.
    unsafe { external_function_call(Some(my_rust_callback)) };
}
