Topic: Setting --no-prepend-enum-name for bindgen?

Alyssa Rosenzweig (Mar 18 2025 at 20:37):

C enum values are not automatically prepended with the enum name, so C code makes a habit of manually prefixing enum values to ensure there are no collisions for the user. This is a common pattern for enums across the kernel. That means if bindgen prefixes with the enum name, we end up with a goofy/verbose double prefix. Not the end of the world, but a very annoying paper cut e.g. when working with uAPI headers.

Alyssa Rosenzweig (Mar 18 2025 at 20:37):

For example, my wip branch of drm/asahi has the following code:

         match data.op {
            uapi::drm_asahi_bind_object_op_DRM_ASAHI_BIND_OBJECT_OP_BIND => {
                 Self::do_gem_bind_object(device, dev_data, data, file)
             }
            uapi::drm_asahi_bind_object_op_DRM_ASAHI_BIND_OBJECT_OP_UNBIND => {
                 Self::do_gem_unbind_object(device, dev_data, data, file)
             }

Alyssa Rosenzweig (Mar 18 2025 at 20:38):

consuming the C enum:

Alyssa Rosenzweig (Mar 18 2025 at 20:38):

enum drm_asahi_bind_object_op {
        DRM_ASAHI_BIND_OBJECT_OP_BIND = 0,
        DRM_ASAHI_BIND_OBJECT_OP_UNBIND = 1,

Alyssa Rosenzweig (Mar 18 2025 at 20:39):

It would be much nicer if we could set --no-prepend-enum-name when calling bindgen. This would let the code be a much more compact / close to the original C enum, uapi::DRM_ASAHI_BIND_OBJECT_OP_BIND

Alyssa Rosenzweig (Mar 18 2025 at 20:40):

Andreas Hindborg (Mar 18 2025 at 20:47):

For hrtimer I just did Rust versions of the enums:

/// Restart policy for timers.
#[derive(Copy, Clone, PartialEq, Eq, Debug)]
#[repr(u32)]
pub enum HrTimerRestart {
    /// Timer should not be restarted.
    NoRestart = bindings::hrtimer_restart_HRTIMER_NORESTART as u32,
    /// Timer should be restarted.
    Restart = bindings::hrtimer_restart_HRTIMER_RESTART as u32,
}

That allows better naming and attaching proper documentation to the variants. Depending on how many enums you are working with, this might be viable for you?
