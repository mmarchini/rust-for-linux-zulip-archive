Topic: ✔ Question about unwrap() on Some()

Joel Fernandes (May 20 2025 at 14:00):

Hello,
I have a question about r4l convention:
if we are guaranteed that an Option is a Some() and not None, can we use unwrap() for readability?
example:

let mut foo_opt = opt;
if let Some(foo) = foo_opt {
    bar = foo_opt.take().unwrap();
}

Otherwise the code becomes ugly with pointless error returning:

let mut foo_opt = opt;
if let Some(foo) = foo_opt {
    bar = foo_opt.take().ok_or(EINVAL)?;
}

bjorn3 (May 20 2025 at 14:37):

In your first code snippet you can write bar = foo; inside the if, right?

Joel Fernandes (May 20 2025 at 14:41):

actually let me show the full code, it is some code I was discussing with Danilo on the list:

        let mut second = second_fwsec_image; // Take ownership of the option
        if let (Some(second_ref), Some(first), Some(pci_at)) =
            (second.as_mut(), first_fwsec_image, pci_at_image)
        {
            second_ref
                .setup_falcon_data(pdev, &pci_at, &first)
                .inspect_err(|e| dev_err!(pdev.as_ref(), "Falcon data setup failed: {:?}\n", e))?;
            Ok(Vbios {
                fwsec_image: second.take().unwrap(),
            })
        } else {
            dev_err!(
                pdev.as_ref(),
                "Missing required images for falcon data setup, skipping\n"
            );
            Err(EINVAL)
        }

Here, I cannot do Ok(Vbios { fwsec_image: *second_ref; }) because the Image type does not have the Copy trait. I believe I will run into the same problem with your suggestion as well. I cannot just do = foo because it will require implementing the Copy trait, which is expensive for this object since it is a large struct. Instead I'd like to do:

   Ok(Vbios {
                fwsec_image: second.take().unwrap(),
  })

which is safe because the Option is guaranteed to be a Some(). Hope that is Ok?

Joel Fernandes (May 20 2025 at 15:08):

Ah Danilo made a nice suggestion which now does not even need the unwrap, since unwrapping is implicit in the Some() expression:

        if let (Some(mut second), Some(first), Some(pci_at)) =
            (second_fwsec_image, first_fwsec_image, pci_at_image)
        {
            second
                .setup_falcon_data(pdev, &pci_at, &first)
                .inspect_err(|e| dev_err!(pdev.as_ref(), "Falcon data setup failed: {:?}\n", e))?;
            Ok(Vbios {
                fwsec_image: second,
            })
        }

So that solves it nicely :)
