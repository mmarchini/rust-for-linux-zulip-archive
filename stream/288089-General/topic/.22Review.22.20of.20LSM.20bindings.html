Hey, for the Rust LSM I created, it would be great if I could get a quick sanity check if my usage of unsafe for the bindings looks correct. These are of course far from production ready, which is why I put review in quotes in the title. At the moment they only allow registering a single LSM directly, which is why I included them with my LSM code, even though it breaks the "only the kernel crate may access the bindings crate" rule. Long term there probably needs to be a macro, similar to how normal module registration works, if someone actually wants to upstream them. For my implementation the current state was enough and I didn't want to spend weeks of thesis time on refining the bindings. I have a branch with only the bindings and a small demo LSM, that just allows every request, which I could use to send an RFC PATCH. Should I just send this to the list? And maybe also to the LSM list, then with a bit more explanation about it?
