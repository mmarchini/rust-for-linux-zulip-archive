Since a few people are curious about the purpose of rust-dev, here is the answer.

It's currently an experimental branch for patch integration, basically a patch queue, potential use cases are:

  1. find merge/apply conflicts as early as possible
  2. provide a common base if one's development relies on multiple things that have not been queued in rust-next.
  3. (maybe) import Rust-related changes from other subsystems if that's helpful.

The reason why there is no public announcement about the branch is simply because of its experimental nature and that its potential use cases can be achieved in different ways: for example, use case 2 can be done by creating one's own branch to track the dependencies and use case 3 can be done by using linux-next branch.

Also the rust-devbranch is volatile and can be rebased/updated frequently, using a snapshot of it makes sense if the work is still in the prototype stage, but for patch submission, please always make it based on rust-next or the next branch of other subsystems (unless maintainers say otherwise).

Hope this helps.
