Topic: Per-call-site data and lock class keys

Gary Guo (Feb 06 2026 at 17:27):

Currently we have a proliferation of macros due to the need of lock class keys.

I have a quite hacky POC that uses inline assembly to attach unique data per call site: https://play.rust-lang.org/?version=stable&mode=release&edition=2024&gist=059879a9f9d629d62f7fd207cc0b1b22.

I think this is too hacky to actually use in the kernel, but I'm throwing it here to invite ideas that would improve the ergnomics when initialization requires lock classes.

Gary Guo (Feb 06 2026 at 17:31):

The deduplication thing is basically to ensure that if a function gets instantiated twice (e.g. into two crates because it's inlinable), the same instance is provided. If we just use the data without deduplication, then we have 2 lock class keys where we want one.
