Topic: dropping the `new_uninit` feature

Benno Lossin (Apr 26 2024 at 06:29):

Since we already have our own BoxExt trait, I was thinking if we could drop the new_uninit feature and just implement it ourselves. Box implements it like this:

    pub unsafe fn assume_init(self) -> Box<T, A> {
        let (raw, alloc) = Box::into_raw_with_allocator(self);
        unsafe { Box::from_raw_in(raw as *mut T, alloc) }
    }

And we could just do that ourselves.

The other functions also look simple enough to do.
