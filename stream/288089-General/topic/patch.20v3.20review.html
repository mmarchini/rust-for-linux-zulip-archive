In the doc comment of https://lore.kernel.org/lkml/20220117053349.6804-6-ojeda@kernel.org/ floating-point functionality is mentioned as an example of unused functionality that needs compiler-builtins intrinsics. This is no longer the case since https://github.com/rust-lang/rust/pull/86048 and those intrinsics are thus not defined anymore. Only 128bit ints and 64bit multiplication/division intrinsics are still defined. The doc comment should probably be updated.

bjorn3 (Jan 17 2022 at 17:37):

In https://lore.kernel.org/lkml/20220117053349.6804-9-ojeda@kernel.org/ extern "C" fn init_module() { ... } and the other extern "C" functions should probably be marked as unsafe. Rustc doesn't automatically mark them as unsafe. That only happens for extern "C" { ... }, declarations, not for definitions. In addition fn __init() { ... } and fn __exit() { ... } should probably be unsafe. They contain unsynchronized accesses to static mut __MOD and would probably invalidate safety assumptions made by functions using __MOD.

bjorn3 (Jan 17 2022 at 17:38):

While it isn't very likely someone will accidentally misuse them I think, it is still technically unsound.
