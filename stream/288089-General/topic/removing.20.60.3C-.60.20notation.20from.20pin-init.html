Topic: removing `<-` notation from pin-init

Benno Lossin (Aug 06 2025 at 10:14):

I'm thinking about removing the <- syntax from the initializer macros in pin-init. Instead one would just use : and the expected type on the right is an initializer. Since all values are initializers via a blanket impl, one can continue to use values on the right side.

It might worsen type inference, I will have to see. The advantage is that now rustfmt will work with pin_init!(Struct { ... }) since it contains valid rust syntax. It's also closer to the language proposal for in-place init by Alice Ryhl.

I've also heard that some people think it's better to have explicit syntax for using an initializer. So maybe we should keep it. Any thoughts on this?
