Colin Braun (Nov 26 2025 at 18:35):

I haven't seen any discussions about the possible addition of macros to generate builder pattern code, which I think could be useful for some structs (this thought crossed my mind while I was tinkering around with adding some usb rust abstractions).

Macros that implement the builder pattern reduce boilerplate code, and are generally more readable and ergonomic. Implementing this with the typestate pattern and generics results in checks done at compile-time with little to no runtime performance cost (see crates like bon and their performance metrics).
