George Vassilev (May 21 2023 at 18:40):

I believe communication across different compiler backends is harder. In this case, they are both made using LLVM, so it's easier to leverage support for Rust when Clang is used instead of GCC. Because GCC is not written with the help of LLVM.

bjorn3 (May 21 2023 at 22:04):

Rust-bindgen is based on libclang and as such only accepts clang C compiler arguments.

bjorn3 (May 21 2023 at 22:06):

kbuild doesn't support passing gcc compiler args when invoking the C compiler while at the same time passing clang args when invoking rust-bindgen. There is some best effort code to filter out gcc args unsupported by clang when invoking rust-bindgen though.

Miguel Ojeda (May 23 2023 at 10:26):

GCC generates pure assembly code

GCC also has its own IRs, just like LLVM.

Is the reason for the use of LLVM in this project only because Rust uses it or why?

No, the kernel can be compiled with GCC and LLVM, thus the Rust support ideally should support both options.

Building the kernel with a full LLVM toolchain is the best supported setup now, but as @bjorn3 points out, we have best-effort support for GCC kernel builds with Rust enabled. For that, the plan is to eventually use rustc_codegen_gcc (the GCC backend of rustc) and/or Rust GCC (the Rust frontend of GCC).
