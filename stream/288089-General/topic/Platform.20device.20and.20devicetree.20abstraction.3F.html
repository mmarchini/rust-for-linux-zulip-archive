Ayush Singh (Sep 07 2024 at 17:37):

Hello everyone. I have recently been working on adding mikrobus addon boards support to upstream kernel. More information about how it will be done can be found in the following patch series.
However, to keep it short, the support is heavily based on device tree and will only have a platform driver stub in the begining. This stub driver will later be expanded to allow dynamic devicetree loading using sysfs entry and reading 1-wire eeprom in boards with it. But the basic board support just needs a stub driver, to allow acess to nexus nodes, and nothing more.
I have also been interested in Rust for Linux and thought that this might be a good driver to upstream platform device and driver abstractions. After going through the recent discussions in the mailing list, it seems like a generic driver registration or id table patch will not be accepted. So, I modified @Fabien Parent patch to remove dependence on the generic abstactions.
I would like it if people more familiar with Rust for Linux can help find any obvious flaws with my patches: https://github.com/Ayush1325/linux/commits/b4/mikrobus-dt/
