Topic: devbound PinInit trait to avoid overuse of Devres

Markus Probst (Nov 19 2025 at 15:06):

I have noticed that a lot of abstractions return impl PinInit<Devres<SomeType>, Error>. This isn't always necessary, as there are places in which it is guaranteed to be dropped as soon as the device is unbound from the driver:

Example 1 (probe in a Bus device):

#[pin_data]
struct SamplePlatformDriver {
    #[pin]
    iomem: Devres<IoMem<0x20>>,
}

impl platform::Driver for SamplePlatformDriver {

    fn probe(
        pdev: &platform::Device<Core>,
        info: Option<&Self::IdInfo>,
    } -> impl PinInit<Self, Error> {
        try_pin_init!(Self {
            iomem <- {
                let request = pdev.io_request_by_index(0).ok_or(ENODEV)?;
                request.iomap_sized::<0x20>()
            }
        })
    }

}

Devres provides no additional guarantee here, as it is dropped anyway as soon as the driver is unbound.

Example 2 (Devres in Devres):
(not a very realistic example, but it fullfils its purpose)

#[pin_data]
struct Data {
    #[pin]
    iomem: Devres<IoMem<0x20>>,
}

impl Data {

    fn new(dev: &platform::Device<Bound>) -> impl PinInit<Devres<Self>, Error> + '_ {
        Devres::new(dev.as_ref(), try_pin_init!(Self {
            iomem <- {
                let request = pdev.io_request_by_index(0).ok_or(ENODEV)?;
                request.iomap_sized::<0x20>()
            }
        })
    }

}

A field with Devres in a struct thats already wrapped in a Devres provides no addition guarantee.

Proposed solution:

Add a new trait DevboundPinInit:

pub unsafe trait DevboundPinInit<T: ?Sized, E = Infallible>: Sized {

    // the implementor needs to validate, that `dev` is the same device as they got passed as parent.
    unsafe fn __pinned_devbound_init(self, dev: &device::Device<Bound>, slot: *mut T) -> Result<(), E>;

    unsafe fn into_pin_init(self, dev: &device::Device<Bound>) -> impl PinInit<Self, Error> + '_ {
        // return a PinInit wrapper that calls self.__pinned_devbound_init
    }

}

unsafe impl<P: PinInit<T, E>, T: ?Sized, E> DevboundPinInit<T, E> for P {

    unsafe fn __pinned_devbound_init(self, _dev: &device::Device<Bound>, slot: *mut T) -> Result<(), E> {
        unsafe { self.__pinned_init(slot) }
    }

    unsafe fn into_pin_init(self, dev: &device::Device<Bound>) -> impl PinInit<Self, Error> + '_ {
        self
    }

}

Change the return value of probe in every bus device from PinInit to DevboundPinInit (as it guarantees to be dropped as soon the driver is unbound).

Change the parameter type in Devres::new from PinInit to DevboundPinInit to still allow Devres to be used, if we want to put in a Box or Arc for example.

Add a new macro try_devbound_pin_init that takes care of implementing the trait:

try_devbound_pin_init!(
    dev,
    < same syntax as try_pin_init! >
)

Example 1 with this solution:

#[pin_data]
struct SamplePlatformDriver {
    #[pin]
    iomem: IoMem<0x20>,
}

impl platform::Driver for SamplePlatformDriver {

    fn probe(
        pdev: &platform::Device<Core>,
        info: Option<&Self::IdInfo>,
    } -> impl DevboundPinInit<Self, Error> {
        try_devbound_pin_init!(pdev.as_ref(), Self {
            iomem <- {
                let request = pdev.io_request_by_index(0).ok_or(ENODEV)?;
                request.iomap_sized::<0x20>()
            }
        })
    }

}

Example 2 with this solution:

#[pin_data]
struct Data {
    #[pin]
    iomem: IoMem<0x20>,
}

impl Data {

    // This can still be wrapped into a Devres
    fn new(dev: &platform::Device<Bound>) -> impl DevboundPinInit<Self, Error> + '_ {
        try_devbound_pin_init!(dev.as_ref(), Self {
            iomem <- {
                let request = pdev.io_request_by_index(0).ok_or(ENODEV)?;
                request.iomap_sized::<0x20>()
            }
        }
    }

}

Advantages

  • more clean driver code
  • no unnecessary devm actions (this shouldn't be an issue).

Disadvantages

  • Adds a lot of complexity to pin-init.

I would like to hear your opinion and thoughts on the topic.

Alternative solution:

trait DevboundPinInit<'a, T, E> {
  fn pin_init(self, token: DevresToken<'_>) -> impl PinInit<T, E> + 'a;
}

impl<P: PinInit<T, E>, T, E> DevboundPinInit<T, E> for P {
  fn pin_init(self, _token: DevresToken<'_>) -> impl PinInit<T, E> + 'a {
    self
  }
}

impl<F: for<'b> FnOnce<DevtresToken<'b>> -> impl PinInit<T, E>, T, E> DevboundPinInit<T, E> for F {
  fn pin_init(self, token: DevresToken<'_>) -> impl PinInit<T, E> + 'a {
    self(token)
  }
}

Devres then:

impl<T: Send> Devres<T> {
    pub fn new<'a, E>(
        dev: &'a Device<Bound>,
        data: impl DevboundPinInit<T, E> + 'a,
    ) -> impl PinInit<Self, Error> + 'a
    where
        T: 'a,
        Error: From<E>;
}

Just to avoid |_token| if we don't need the token.
