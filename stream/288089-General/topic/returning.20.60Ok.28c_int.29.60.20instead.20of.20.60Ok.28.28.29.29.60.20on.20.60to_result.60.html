Subject: [OLD PATCH] rust: error: return value in `to_result`'s successful path

`to_result` is used to conveniently convert a Linux kernel error
code (an `int`) into a Rust `Result` with no value returned in
the successful case.

In some cases, C APIs return error-or-value integers, i.e. the value
is meaningful and needed afterwards. At the moment, if one wants to
take advantage of `to_result` in this case, one may write code like:

    // SAFETY: ...
    let ret = unsafe { bindings::...() };
    to_result(...ret...)?;
    // Use `ret`.

Returning the error back to the caller in the happy path, i.e. within
the `Ok` variant, allows to shorten code a bit in those cases:

    // SAFETY: ...
    let value = to_result(unsafe { bindings::... })?;

It also encourages to give meaningful names for the local (instead
of generic names like `ret`), since the variable at that point is
not anymore a error-or-value, but the actual integer we need.

In addition, the fact that the returned value is non-negative can
be encoded in the type system, returning an unsigned integer.

Moreover, for all kernel architectures, `sizeof(int) == 4`, so
assuming that the kernel will always be LP64 and that there will be
no ILP128 kernel code, `u32` can be returned. Just in case, add
a static assertion for the future.

Thus make `to_result` return an `u32` with the value.

That's another limitation of to_result. It only supports i32, so we have to cast other types to i32 in order to use to_result, and even that isn't always possible in every case (e.g., see https://lore.kernel.org/all/20250822080252.773d6f54@nimda.home/).

So in addition to returning the given number, we could make it generic to pass any type of number on to_result and it will the same type back, so things like Result<u16> (and https://lore.kernel.org/all/20250822080252.773d6f54@nimda.home) can work.

A quick/rough to_result P.o.C would look something like this:

pub type Result<T, E = ()> = core::result::Result<T, E>;

pub fn to_result<T>(v: T) -> Result<T>
where
    T: PartialOrd + Default + Copy,
{
    if v < T::default() { Err(()) } else { Ok(v) }
}

fn main() {
    let x = to_result(1i32).unwrap();
    let y = to_result(5i16).unwrap();
    let z = to_result(10i8).unwrap();
    let t = to_result(42u32).unwrap();
}

(It could probably be implemented in a much better way, I don't have time to think about it in more detail today.)
