Would it be possible to mark functionality which is not allowed to be called from interrupt context with some kind of "noirq type"? Similar to what Benno does with the untrusted data, but instead of marking data untrusted mark the functionality as "not allowed to be executed in irq context".

In this specific case I wonder if it would be possible to mark the timer callback as an "interrupt context function" and the mutex as "not allowed to be called from interrupt context". And with that make the compiler fail on it.

Would that be possible?
