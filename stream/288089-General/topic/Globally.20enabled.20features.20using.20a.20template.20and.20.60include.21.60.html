bjorn3 (Sep 28 2021 at 06:18):

Rustc has the unstable #[prelude_import] attribute you can apply to a wildcard export. This is how internally the prelude works. It only works for the current crate though. You can't set a global prelude.

AFAIK #[prelude_import] causes the use to be in scope for every module, not just the one containing the use.

bjorn3 (Sep 28 2021 at 20:57):

If I macro expand a crate containing only #![no_std] I get:

#![feature(prelude_import)] #![no_std]
#[prelude_import]
use core::prelude::rust_2018::*;
#[macro_use]
extern crate core;
#[macro_use]
extern crate compiler_builtins;

a completely empty crate gives:

#![feature(prelude_import)]
#[prelude_import]
use std::prelude::rust_2018::*;
#[macro_use]
extern crate std;
