Hi all,
I previously worked on a pointer formatting patch for the kernel fmt module (last submitted version: https://lore.kernel.org/rust-for-linux/20260311-hashedptr-v11-1-dd1b8e8b0fb0@kylinos.cn/).

I learned a lot from all the valuable feedback everyone gave on earlier versions. I've made another round of improvements recently, and I'd like to get some early review suggestions before submitting the next formal version.

The latest version ready for review is at:
https://github.com/alvinsun039/linux-torvalds/commit/eee0c7e107042a98914b47a6f0d99fd05145f5ac

Key changes:

  • Add T: ?Sized bound to all Pointer trait implementations, combined with standard cast::<()>() type conversion in HashedPtr to uniformly extract pointer data address and automatically discard metadata, adding support for {:p} formatting of all fat pointer types
  • Add Pointer trait implementation for mutable references
  • Add zero-padding format support
  • Refactor test suite: add RAII state guard, split into independent test cases, add fat pointer equality validation tests
    Any comments, suggestions or review feedback are highly appreciated!

Thanks,
Alvin Sun
