Asahi Lina (Feb 09 2023 at 03:31):

The stack leak one is the same problem I have with sharing structures across trust boundaries, and there are other fundamental issues like trying to read data from userspace into Rust structures where not all bit patterns are valid. This should be solvable with the Castable idea that I plan to start using, which is a trait that guarantees that the struct has no padding and all bit patterns are valid. We should probably require that for all copies to/from userspace...

CONFIG_INIT_STACK_ALL_ZERO is really just a band-aid for the fact there's no way to enforce this in C's type system, but we can do better than that ^^

Asahi Lina (Feb 09 2023 at 03:37):

The buffer race one is just trying to port C bugs to Rust as far as I can see. IoBufferReader.read_raw is unsafe and that's where all the overflow problems come from (the code looks like C ported 1:1 to Rust). They should just be using IoBufferReader.read_slice instead. And then the Vec version correctly panics instead (which of course bad Rust code can)...

Asahi Lina (Feb 09 2023 at 03:40):

The pointer leak issue is real though, I don't think there's any good way of truly stopping that with Rust's current type system (since getting raw pointers out and casting them to usize is safe), but we should at least figure out how to enable pointer masking in format strings for pointer types...
