Benno Lossin (Apr 18 2024 at 10:10):

the error that Box::try_new returns is AllocError. this means that if it fails, it must be becausr there is no memory left. the general Error type (that declares the constant ENOMEM) also can represent othrr failures such as EINVAL.

Benno Lossin (Apr 18 2024 at 10:12):

so yes, it is intentional. it also allows for better optimizations, since Result<Box<T>, AllocError> is pointer sized.

Alice Ryhl (Apr 18 2024 at 10:50):

Another way to go about it is this:

let b = match Box::try_new(foo) {
    Ok(b) => b,
    Err(AllocError) => {
        clean_up();
        return Err(ENOMEM);
    },
};

By typing Err(AllocError), you are asserting that Box::try_new returns an AllocError, and the code will not compile if there are other possible reasons Box::try_new could fail.

I've sometimes seen comments along the lines of "// this function can only fail with ESRCH", and this pattern allows you to encode that assertion as a checked condition in your code. As a similar example, see BadFdError from my file patchset. By using the same syntax, you can assert in your code that an fget call can only fail with EBADF and that no other error conditions exist for that method. I think that is useful.
