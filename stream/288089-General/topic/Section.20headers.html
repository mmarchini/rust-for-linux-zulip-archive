Topic: Section headers

Benno Lossin (Sep 10 2024 at 17:31):

In Danilo's Allocator patch series, I stumbled on the section ordering Safety and then Examples, I also had this come up in the past and I had the following response then:

I think we should standardize that Invariants and Safety sections
are put last. This is because people reading the HTML version are
probably not interested in the inner workings. This also makes it
possible to have the invariants and the struct definition on the same
screen.

I want to create a good-first-issue for this. How do we want to sort the other section headers? My initial idea is this:

  • # Examples
  • # Panics
  • # Invariants
  • # Guarantees
  • # Safety

I also saw that in the old rust branch we had the Errors section header (we also have some others) from vendored alloc, do we want to "revive" that? (ie use it to document when a function returns Err)

Benno Lossin (Sep 10 2024 at 17:33):

The idea for doing this is the following:

  • make safety documentation immediately visible for people viewing the source code (eg when using an LSP to jump to the definition)
  • for people browsing the HTML or viewing the documentation via an LSP, the Examples section comes first, so it's easy to get started with the API
