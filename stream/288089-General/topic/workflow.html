Either way, please note that the rust branch has been obsolete for months now (please see https://rust-for-linux.com/branches#rust).

We understand that over time we got some downstream users, and we appreciate all the interest in the Rust support.

But please do not base work on top of rust. It is not supported, it contains APIs that have been reworked substantially since then (e.g. all types using pinned-init), it contains experimental/WIP code, etc.

And since some APIs have been reworked w.r.t. mainline, in order to keep your code working, we would need to keep all the old code/types/APIs as-is. And even if we did that, we would not know if we would have successfully kept your code working.

Miguel Ojeda (Aug 19 2023 at 18:29):

(Please format your messages properly, it is fairly hard to read otherwise.)

I mean rebase rust branch on the linux'tree or rust-next . which let some code which have not been upstreamed to linux-tree such as the rust/kernel/fs.rs code to be applied to the linux'tree head or rust-next head.

Rebasing would give you the same contents as the merges I performed in the past. Anyway, not all the code in rust is suitable or intended for upstreaming as it is, for different reasons.

If we just rebase our rust branch on the linustree or rust-next, then we will no need cherry-pick/sync commits from linux tree or new commit just match the linus tree's, such as the commit "rust: upgrade to Rust 1.71.0".

The rebasing you are asking for is the upstreaming effort that we are doing, for those use cases where we have a user. This involves cleaning things up, re-reviewing them to catch design/soundness/... issues if any, writing proper commit messages, splitting dependencies, talking to other kernel maintainers, and so on and so forth. It is not as simple as "just rebase".

Therefore, if you have a downstream user that you can submit upstream, then please let us know and we will try to help you getting your dependencies and your user there.

If you do not intend to upstream a user to the kernel, then please note that rust branch is not supported, that it is going away soon, and that some of its contents may never be upstreamed. That is why we need use cases in-tree: in order to upstream things.
