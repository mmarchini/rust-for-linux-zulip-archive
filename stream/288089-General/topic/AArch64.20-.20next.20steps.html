Miguel Ojeda (Feb 14 2024 at 11:06):

It shall not be an existing C driver or similar just ported to Rust.

It can be an existing C driver if the subsystem maintainers agree to have a "Rust reference driver" to bootstrap things (i.e. a typically small but well-known driver in their subsystem).

For instance, see the Asix PHY driver that is now on mainline:

config AX88796B_RUST_PHY
    bool "Rust reference driver for Asix PHYs"
    depends on RUST_PHYLIB_ABSTRACTIONS && AX88796B_PHY
    help
      Uses the Rust reference driver for Asix PHYs (ax88796b_rust.ko).
      The features are equivalent. It supports the Asix Electronics PHY
      found in the X-Surf 100 AX88796B package.

It allows you to switch (at build time) between the C or the Rust driver.

To be clear, that kind of exceptional driver is an exceptional case and is only meant for bootstrapping -- duplicate code is not allowed in the kernel in general, so one of the two drivers should eventually go away.
