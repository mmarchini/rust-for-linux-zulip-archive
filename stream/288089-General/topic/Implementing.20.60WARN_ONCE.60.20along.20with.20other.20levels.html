Topic: Implementing `WARN_ONCE` along with other levels

Onur Özkan (Jul 09 2025 at 12:17):

We have two TODOs related to adding WARN_ONCE support in the Rust side of the kernel tree:

// rust/kernel/mm/virt.rs
// TODO: call WARN_ONCE once the Rust version is available

// rust/kernel/error.rs
// TODO: Use `WARN_ONCE` once it's implemented

I am thinking to implement this along with support for other log levels (e.g., info, notice, emerg, etc.).

Right now, I am considering two possible approaches:

  1. Adding *_once macros for each log level:
pr_info_once!("Some info, just once");
  1. Extending the existing pr_* macros to optionally support once logging:
pr_info!("Some info");              // Standard usage
pr_info!("Some info", once = true); // Log once

I would be happy to hear any opinions on this, whether the first or second approach is better or if there is a better alternative altogether.
