Martin Rodriguez Reboredo (Nov 11 2023 at 13:37):

Something interest that I've found is that you can compile a library that only contains reexports and it won't produce symbols, only an empty object and the rmeta. This is useful in case of having various crates living under a subsystem but you want them to be all-in-once.

Martin Rodriguez Reboredo (Nov 11 2023 at 13:39):

For example...

// drivers/usb/core/usb_core.rs
pub fn disabled() -> bool {
    bindings::usb_disabled() != 0
}

// drivers/usb/lib.rs
pub use usb_core::disabled;

Martin Rodriguez Reboredo (Nov 11 2023 at 13:47):

You can have a crate that does reexports and you don't have to link it to anything, you only require its metadata.
