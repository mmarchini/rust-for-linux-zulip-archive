Topic: Clist rust abstraction

Joel Fernandes (Oct 16 2025 at 01:08):

I started to implement a Rust module for the kernel crate called clist. The idea is that it will allow to perform list operations on lists that were allocated and created on the C side but now need to be processed on the rust side. I need this for DRM buddy allocator rust bindings. The C side allocates a list of blocks and creates a C-side circular doubly linked list out of them. On the rust side, we have to iterate over the list and call C side bindings on the list_head of different blocks. The clist module will also provide bindings to common C list helpers like list_del and list_empty.

Just wanted to share the idea here as I work on it. Looking forward to any initial comments on the idea here.
