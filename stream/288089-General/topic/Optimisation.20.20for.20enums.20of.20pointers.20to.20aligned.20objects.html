Topic: Optimisation for enums of pointers to aligned objects

Andreas Hindborg (Jan 23 2026 at 15:31):

Optimization opportunity for pointers to aligned objects

Consider the following Rust (Linux kernel) code:

enum Node {
    Pointer(KBox<Node>),
    Internal(u8),
    Value(u8),
}

The size of Node is 2 words, one for the discriminant and one for the value.
However, because Node has an alignment of at least 4 bytes, the discriminant could
be encoded in the lower two bits of the first word.

This is an optimization that is applied by hand in key areas of the Linux kernel
C code to achieve lower memory pressure and better cache utilization.
Specifically, the Linux kernel's C xarray uses this pattern.

For the Linux kernel's XArray, this means that an idiomatic Rust implementation
uses twice the memory of the C implementation. Matching the memory density of
the C implementation with Rust code requires manual pointer tagging with unsafe
code.

The effort to implement the Linux kernel xarray in Rust is driven in part by
memory safety issues encountered in the existing C implementation. Thus, having
to reach for unsafe blocks for cases like this is very undesirable.

Use Case: Linux Kernel XArray

The XArray is a kernel data structure that acts as a key/value map, mapping integers
to object addresses, or integers to integers. It's used in the page cache,
filesystems, and block layer.

Pointers stored in XArray must be 4-byte aligned (true for kmalloc() and
alloc_page() returns). Integers between 0 and LONG_MAX can be stored via
xa_mk_value().

The XArray is a radix tree with a default shift value of 6 such that each node
stores 64 pointers to the next level of the tree. In C, with 8-byte tagged
pointers, this results in 512-byte nodes. An idiomatic Rust implementation
requires 16 bytes per entry (8 for discriminant + 8 for value), resulting in
1024-byte nodes. The performance impact of this difference is significant.

Conclusion

There exists an optimization opportunity for Rust enums that contain pointers to
aligned objects. Unlocking this optimization is key in achieving a performant
Rust XArray implementation without unsafe blocks.

use std::mem::{self, ManuallyDrop};
use std::ptr;

// Assuming KBox is an alias for Box or a similar kernel smart pointer.
// We use Box here for demonstration.
pub type KBox<T> = Box<T>;

#[repr(transparent)]
pub struct PackedNode {
    ptr: *const Node,
}

pub enum Node {
    Pointer(KBox<Node>),
    Internal(u8),
    Value(u8),
}

impl PackedNode {
    const TAG_MASK: usize = 0b11;
    const TAG_PTR: usize = 0b00;
    const TAG_INT: usize = 0b01;
    const TAG_VAL: usize = 0b10;

    pub fn new_pointer(kbox: KBox<Node>) -> Self {
        let ptr = KBox::into_raw(kbox) as *const Node;
        debug_assert_eq!(ptr.addr() & Self::TAG_MASK, Self::TAG_PTR);
        Self { ptr }
    }

    pub fn new_internal(val: u8) -> Self {
        let addr = ((val as usize) << 2) | Self::TAG_INT;
        let ptr = ptr::without_provenance(addr);
        Self { ptr }
    }

    pub fn new_value(val: u8) -> Self {
        let addr = ((val as usize) << 2) | Self::TAG_VAL;
        let ptr = ptr::without_provenance(addr);
        Self { ptr }
    }

    pub fn unpack(self) -> Node {
        let this = ManuallyDrop::new(self);
        let ptr = this.ptr;
        let tag = ptr.addr() & Self::TAG_MASK;

        match tag {
            Self::TAG_PTR => unsafe {
                let kbox = KBox::from_raw(ptr as *mut Node);
                Node::Pointer(kbox)
            },
            Self::TAG_INT => {
                let val = (addr >> 2) as u8;
                Node::Internal(val)
            }
            Self::TAG_VAL => {
                let val = (addr >> 2) as u8;
                Node::Value(val)
            }
            _ => unreachable!(),
        }
    }

    pub fn is_pointer(&self) -> bool {
        (self.ptr.addr() & Self::TAG_MASK) == Self::TAG_PTR
    }
}

// Drop implementation is critical to prevent memory leaks for the KBox variant.
impl Drop for Node {
    fn drop(&mut self) {
        if self.is_pointer() {
            unsafe { drop(KBox::from_raw(self.ptr as *mut Node)) };
        }
    }
}

unsafe impl Send for PackedNode where Node: Send {}
unsafe impl Sync for PackedNode where Node: Sync {}

A quick sketch:

enum Unpacked<T: ForeignOwnable> {
    Pointer(T),
    Tagged {
        tag: Bounded<NonNull<usize>, {align_to_bits(T::FOREIGN_ALIGN)}>,
        value: Bounded<usize, {usize::BITS-align_to_bits(T::FOREIGN_ALIGN)}>,
    },
}

struct Packed<T: ForeignOwnable> {
    packed: *const c_void,
}
