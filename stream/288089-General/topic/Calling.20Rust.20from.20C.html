Miguel Ojeda (Dec 14 2023 at 18:26):

Julian Stecklina said:

I was wondering whether it's supported to build Rust libraries in the kernel that are then consumed by C code. (...) Or is calling Rust from C code frowned upon in the RfL project?

It is definitely something we would love to see happen (and some maintainers have been asking about it!). However, in the beginning, most likely any such effort will need to be opt-in/parallel (i.e. it could potentially be upstreamed, but most likely would be under a config gate nowadays) due to the current constraints (like the version and platform support) to avoid requiring Rust for every user. It would also be nice to provide some facilities to help with that too.

But if you want to experiment for the moment, then you can already start by putting your code in the kernel crate, manually writing the declarations on the C side and compile KVM as built-in (if that does not work for some reason, please feel free to send me an email and I am happy to help).

In fact, if you get that project rolling (and especially if you manage to show its value to the maintainers), it could serve as a justification for introducing the extra support needed to do things properly. So please let us know if you do start working on it!
