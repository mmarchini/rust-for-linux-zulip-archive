Topic: Result::to_errno helper

Alyssa Rosenzweig (Apr 14 2025 at 21:20):

While reviewing the DRM abstractions, I noticed a pattern:

match x {
    Err(e) => e.to_errno(),
    Ok(()) => 0
}

This seems to come up in other abstractions too. Time for a helper for this..?

Alyssa Rosenzweig (Apr 14 2025 at 21:23):

This should compose with map as well, the DRM code on-list has

        let inner = match T::open(drm) {
            Err(e) => {
                return e.to_errno();
            }
            Ok(i) => i,
        };

        let driver_priv = KBox::into_raw(unsafe { Pin::into_inner_unchecked(inner) });
        unsafe { (*file.as_raw()).driver_priv = driver_priv.cast() };

        0

which with this helper could instead be something like:

T::open(drm).inspect(|inner| {
        let driver_priv = KBox::into_raw(unsafe { Pin::into_inner_unchecked(inner) });
        unsafe { (*file.as_raw()).driver_priv = driver_priv.cast() };
}).to_errno()

One small ergonomic painpoint is that Result comes from the stdlib, so we can't just define methods for it, we need an extension trait instead. So we could do:

pub trait IntoErrno {
    fn into_errno(self) -> i32;
}
impl IntoErrno for Result {
    fn into_errno(self) -> i32 {
        match self {
            Ok(()) => 0,
            Err(e) => e.to_errno(),
        }
    }
}
