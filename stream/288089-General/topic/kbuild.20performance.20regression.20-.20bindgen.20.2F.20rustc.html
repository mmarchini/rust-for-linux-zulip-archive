Giuliano Procida (Jun 23 2025 at 13:40):

Hi. First post. I work in the Android Systems team.

Rust support was added with the Android (16) 6.12 kernel. We noticed significant build time regressions compare with Android (15) 6.6. A lot of this was due to gendwarfksysms and was mostly resolved. However, we still have an overall bump in complication times.

Early in the build, only bindgen and then rustc seem to be running and they take a fair while. Once they complete, parallel make explodes across all cores as usual.

Has anyone else noticed or investigated this bottleneck? Is there a false build dependency?

Miguel Ojeda (Jun 23 2025 at 15:20):

Once they complete, parallel make explodes across all cores as usual. (...) Has anyone else noticed or investigated this bottleneck?

This is expected (it runs in an early phase of the build, on purpose) and it has been the case since the very fist time Rust was merged into the kernel (6.1), so it isn't a regression. It will be improved with the future build system changes.
