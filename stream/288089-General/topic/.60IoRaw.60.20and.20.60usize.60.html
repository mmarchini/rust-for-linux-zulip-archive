Danilo Krummrich (Apr 18 2025 at 05:23):

@Andrew Ballance

Cc: @Alice Ryhl, @Rob Herring

Thanks for picking this up!

I briefly went over the patches and there's one major concern from my side:

I don't think in Rust we should ever use the iowrite / ioread function family. The reason is that they solve a problem in C, which in Rust we simply do not have.

In C we only get a raw pointer from ioremap() and hence either I/O accessors have to ensure to do the correct thing in terms of MMIO or PIO, or the driver has to take care by choosing the correct one itself.

In Rust, however, we get an object instance (i.e. Io) from ioremap() functions (such as pci::Device::iomap_region()) and hence can encode whether it is PIO or MMIO from the get-go and subsequently always use the correct accessor functions without redundant checks.

In PCI you can for instance do this by checking pci_resource_flags() for either IORESOURCE_IO or IORESOURCE_MEM (see also [1]) once when things are mapped and subsequently use read / write or in / out accessors.

This is much better than using the ioread / iowrite ones, which are unnecessary wasteful by repeating this check for every I/O access (for instance with CONFIG_GENERIC_IOMAP) [2].

Besides that, I think the patches need to be broken up a bit better. For instance, rust: helpers: io add support for pio accessor functions does not only add new helpers, but also changes existing ones, etc. Such changes should be separate patches.

[1] https://elixir.bootlin.com/linux/v6.14.1/source/drivers/pci/iomap.c#L46
[2] https://elixir.bootlin.com/linux/v6.14.1/source/lib/iomap.c#L52

@Andrew Ballance

This would still leave drivers with the need to convert the Iointo MMIo or PortIo, which I think isn't ideal. We also need to consider that busses wrap this type into a Devres.

I think it would be better to have a common trait for both MMIO and PIO and just let Io determine whether it's MMIO or PIO on creation and put in the correct accessor backend. This way it would be transparent to drivers whether it's MMIO or PIO without any unnecessary, redundant checks.
