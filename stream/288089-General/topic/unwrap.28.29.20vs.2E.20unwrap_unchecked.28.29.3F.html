Topic: unwrap() vs. unwrap_unchecked()?

Lyude (Apr 27 2026 at 18:54):

Hi! Recently in a patch review that I received (the patch review from Alexander on my shmem series in particular) and I noticed that some of the suggested code actually used unwrap() vs. unwrap_unchecked():

// PANIC: `populate` has just succeeded, guaranteeing that `sgt_res` is populated.
Ok(sgt_res.as_ref().unwrap())

Where previously this would have used unwrap_unchecked(), in part because the actual assignment that populated the variable was on the line directly above the unchecked unwrap.

Do we have an explicit rule for when to use one or the other?

FWIW, I've always preferred unwrap_unchecked() for locations where the actual Some() assignment is guaranteed by a type invariant, or the assignment happens immediately above the unchecked unwrap in the source file. And I've usually explicitly avoided unwrap(). I can see some situations where I would possibly want to use unwrap() if I expect it to be possible that a type invariant could be violated on the C side, but that's about it.
