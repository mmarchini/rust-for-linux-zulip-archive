Miguel Ojeda (Aug 05 2024 at 00:15):

What we currently do is provide a weak symbol that panics. It is not great at all. First of all, because we manually list the symbols, which isn't great. But we should not even have those calls being generated to begin with (unless really some code needs them -- very little code needs it, and it is compiled differently in the C side).

Ideally, we would have a compiler flag to disable all that where not needed (possibly even not accepting code referring to f32 etc.). I think @Gary Guo took a look at some point but, while disabling the formatting bits was easy, the rest would not be so much.

Still, it would be nice to get rid of the hack we have. Or at least reduce it a bit (e.g. can we avoid listing them one by one?).
