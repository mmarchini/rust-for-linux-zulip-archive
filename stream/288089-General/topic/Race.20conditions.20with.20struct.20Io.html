Topic: Race conditions with struct Io

Daniel Almeida (Nov 25 2025 at 16:55):

Reading and writing with struct Io only requires a shared reference, as we perform the actual operations using FFI. We should ideally change define_write to take a &mut, or otherwise there is no protection against race conditions when using these functions.

IMHO, this design should be extended to any other write operation, including dma_write and others as applicable.

Thoughts?

Alice Ryhl (Nov 25 2025 at 17:38):

I don't think &mut is the correct protection against race conditions here. We need the ability to perform parallel access to non-overlapping regions, and using &mut for data races fundamentally throws away that capability.
