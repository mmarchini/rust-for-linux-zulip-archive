Topic: Implementing ARef support for structs like `Workqueue`

Lyude (Nov 10 2025 at 21:27):

A recurring problem I've been noticing that we keep running into w/r/t the DRM bindings is the subclass pattern that DRM interfaces use. For instance:

drm::Device<T: drm::Driver>

drm::Driver has an associated type that represents the driver's private data, Data, which can be accessed through Deref<T::Data>. The problem is it's pretty common for a driver to want to be able to store things like workqueues in Data, but because we can't implement WorkItem for drm::Device. The thing is though, I've seen this issue come up in pretty much every subclassed type like this in DRM.

With all of this being said - there's no normal case where an object which implements AlwaysRefCounted and does not embed some sort of data in itself would actually need to use workqueues. It's pretty much exclusive to implementors with embedded data, which is why I actually think it would make sense to introduce a new trait: ARefContainer.

The basic idea is that any type that implements AlwaysRefCounted and has user-specific data embedded in itself (T::Embedded) can implement ARefContainer which provides methods for going from *mut T -> *mut T::Embedded and vice-versa. There, we can start implementing workqueue support for types implementing ARefContainer among other things in the future like HrTimer. Does this seem like a good idea?

use crate::workqueue::HasWork;
use crate::workqueue::Work;
use crate::workqueue::WorkItem;

impl<T, const ID: u64> WorkItem<ID> for Device<T>
where
    T: drm::Driver,
    T::Data: WorkItem<ID, Pointer = ARef<Device<T>>>,
    T::Data: HasWork<Device<T>, ID>,
{
    type Pointer = ARef<Device<T>>;

    fn run(ptr: ARef<Device<T>>) {
        T::Data::run(ptr);
    }
}

unsafe impl<T, const ID: u64> HasWork<Device<T>, ID> for Device<T>
where
    T: drm::Driver,
    T::Data: HasWork<Device<T>, ID>,
{
    unsafe fn raw_get_work(ptr: *mut Self) -> *mut Work<Device<T>, ID> {
        unsafe {
            let data_ptr = &raw mut (*ptr).data;
            T::Data::raw_get_work(data_ptr)
        }
    }

    unsafe fn work_container_of(ptr: *mut Work<Device<T>, ID>) -> *mut Self {
        let data_ptr = unsafe { T::Data::work_container_of(ptr) };
        unsafe { crate::container_of!(data_ptr, Self, data) }
    }
}
