As far as I know there is currently no list like that, sorry. In general, only patches that are required by some user (a driver) are accepted into the kernel. You could look into implementing a driver for some peripheral you use and then the dependencies of that driver will give you a task list.

Andreas Hindborg (Feb 23 2023 at 08:31):

I guess you could check out the issue list on github, but atcjes for the issues will only be accepted into the kernel if there is a user. Also, as far as I understand, our github no longer accepts pull requests. Patches should go to the relevant kernel mailing list. That is rust-for-linux@vger.kernel.org for core Rust things. @Miguel Ojeda please correct me if I'm wrong.
