Topic: Introducing a proc-macro for linking-back to kerneldocs?

Lyude (Mar 04 2025 at 23:18):

Hi! I've been working on getting the rust KMS bindings ready for another iteration to be sent onto the mailing list and there's still one open question I haven't answered - which is linking back to prior written documentation on the C side of things.
The reason this is desireable for KMS is that there's a lot of pretty complicated concepts in KMS that warranted us writing some pretty in-depth documentation in kerneldoc, but the only way I have been able to find to link back to this documentation is using raw kernel.org links which seems less than ideal, as those links tend to be quite big (for example: https://www.kernel.org/doc/html/latest/gpu/drm-kms-helpers.html#c.drm_atomic_helper_fake_vblank ).
Which brings me to ask: has anyone considered trying to automate this partly with a procedural macro in rust? I would think we could basically just write up a proc-macro that you can add as an attribute above an item that just says which C function this should link back to on the kernel doc side, and then automatically insert said link into the rustdoc for that item.

Miguel Ojeda (Mar 04 2025 at 23:24):

Please see the first item in https://github.com/Rust-for-Linux/linux/issues/350 -- my plan has been to generate a map of references from kerneldoc, and then give it to rustdoc (with special support), which then we could simply use as intra-doc links, e.g. writing

See [`struct foo`].

should work ideally (or very similar syntax).
