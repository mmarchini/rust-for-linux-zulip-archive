Topic: doctest

Andreas Hindborg (Jul 07 2022 at 12:28):

In regular rust one can return Result from a doctest, but that is not supported in rust-for-linux. Is anybody planning on adding support for returning Result from doctest?

Miguel Ojeda (Jul 07 2022 at 12:38):

What we do is wrap the test into a block like:

/// ```
/// # use kernel::{spawn_work_item, workqueue};
///
/// # fn example() -> Result {
/// spawn_work_item!(workqueue::system(), || pr_info!("Hello from a work item\n"))?;
/// #     Ok(())
/// # }
///
/// # example().unwrap()
/// ```

There is nowadays support in rustdoc to do this automatically for userspace tests, and I plan to have something similar too. For the moment, please write them like that, and I will clean them up when we have support for it all at once.
