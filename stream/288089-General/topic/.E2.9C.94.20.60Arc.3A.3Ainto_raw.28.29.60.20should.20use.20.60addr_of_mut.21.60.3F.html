Topic: ✔ `Arc::into_raw()` should use `addr_of_mut!`?

Boqun Feng (Jan 06 2025 at 16:40):

Code here:

    pub fn into_raw(self) -> *const T {
        let ptr = self.ptr.as_ptr();
        core::mem::forget(self);
        // SAFETY: The pointer is valid.
        unsafe { core::ptr::addr_of!((*ptr).data) }
    }

If I understand pointer provenance correctly, pointers derived from addr_of!() don't have the provenance for writing, therefore the result of Arc::into_raw() cannot be used to construct an Arc via Arc::from_raw(), right?

Alice Ryhl (Jan 06 2025 at 17:08):

The docs say:

If the place expr evaluates to is based on a raw pointer, then the result of addr_of! inherits all permissions from that raw pointer.
