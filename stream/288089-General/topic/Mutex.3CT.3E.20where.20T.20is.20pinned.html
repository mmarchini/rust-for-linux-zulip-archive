Topic: Mutex<T> where T is pinned

Daniel Almeida (Jun 13 2025 at 13:21):

This appears to not be supported. In other words, it's impossible to have a mutex (which needs to be pinned) wrap something that also needs to be pinned.

This makes the following pattern impossible to implement:

Mutex<Opaque<bindings::foo>> where bindings::foo contains e.g.: a list_head or similar.

Having a mutex wrap a bindgen type might be handy if we want to enforce that all access to the bindgen field be serialized, because it makes it impossible to get a *mut Foo otherwise.

A similar (but much worse) solution would be:

#[pin_data]
struct Foo {
  #[pin]
  inner: Opaque<bindgen::foo>,
  #[pin]
  protects_inner: Mutex<()>
}

That is clearly inferior, as we now must manually make sure that protects_inner is locked before accessing inner, instead of relying on the type system.
