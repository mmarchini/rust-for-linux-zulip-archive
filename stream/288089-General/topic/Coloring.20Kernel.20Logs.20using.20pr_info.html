Topic: Coloring Kernel Logs using pr_info

Mortadha Dhahri (May 29 2024 at 08:22):

Hello everyone , I am trying to costumize the output of my kernel modules using pr_infos and I couldn't help but notice that there are some important warnings and error messages that need different colors to accentuate their importance , is there a way to make those kernel Logs more personalized ? Thank you.

Fabien Parent (May 29 2024 at 17:01):

You can usually get colorized output by running dmesg -L=always, you will get error message in red, warnings in bold. In most case it should be "on" by default but you can check the dmesg manpage to read the details.
Some implementations of dmesg on embedded device might not implement colorization by default to keep the binary small.
I think that it would not be possible to upstream code where you embed ASCII escape code for formatting since it does not play nice when you want to log the output.
