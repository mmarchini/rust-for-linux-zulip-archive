Hey everyone,

for my upcoming master thesis (or as part of it) I wanted to work on a Rust-based Linux Security Module (LSM), so I thought before I go ahead and start that I'd ask here for a few opinions :)

From my brief look into LSMs and the Rust kernel module registration code, it seems like LSM registration would be somewhat similar, mostly with different #[link_section] attributes. I also saw that @Alice Ryhl has sent some patches for bindings to the security/LSM header for Rust Binder, but those look not very LSM specific yet, so these bindings would also be part of the thesis for me.

Do you think creating (not yet production ready) bindings and a very basic Rust LSM on top of them would be feasible as part of a thesis, or is the Rust support in the kernel not there yet? Is anyone already working on LSM bindings?

Miguel Ojeda (Nov 22 2024 at 11:53):

It is nice to see a MSc thesis on Rust for Linux!

I agree that "landing it upstream" as a concrete thesis goal is something that I would generally recommend avoiding, since it depends on whether there is an actual user needing it, on upstream maintainers, etc., unless you have talked to the maintainers first and everyone is in agreement. Hello, very late update from me, my thesis was finally published last week, it's available at https://doi.org/10.22032/dbt.66733. Code and artifacts are available at https://doi.org/10.5281/zenodo.17765425, but more conveniently at https://github.com/o1oo11oo/thesis-code (supporting code) and https://github.com/o1oo11oo/linux/tree/thesis-main (kernel code).

I wanted to fix some typos and oversights before publication, which was allowed, but then I was busy with other things, so even just going through everything once took quite a while unfortunately.

I'm not sure how useful this will be for anyone, as the thesis focuses on dynamic security policies a lot more than the concrete Rust implementation of them, but maybe this is helpful or interesting to someone. I'm not sure if it still makes sense to send this to a mailing list, maybe Rust-for-Linux and LSM? Feedback on this appreciated. The bindings are still in the same state as they were when I sent them to the lists for feedback, and I'm not currently working on polishing them.

The code is as used during the thesis, and especially during the end when I didn't have that much time left I might have cut some corners :) Rebasing this to a more recent kernel version likely requires cleaning that up, but there shouldn't be that much.
