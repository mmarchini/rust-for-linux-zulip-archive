Topic: aarch64 and disabling target features

Alice Ryhl (Jul 04 2024 at 11:26):

The aarch64 target we're using by default enables these target features: +v8a,+strict-align,+neon,+fp-armv8.

But then, we disable the neon feature from the list.

Now, my question is, aren't we supposed to also disable the fp-armv8 target feature?

Miguel Ojeda (Jul 04 2024 at 11:58):

Maybe there was a reason for that -- Cc'ing Jamie via email (he does not seem to be here).

Jamie Cunliffe (Jul 04 2024 at 13:36):

Rust decided to remove the fp target feature and tie it with the neon feature so disabling neon will also disable that feature (https://github.com/rust-lang/rust/blob/master/compiler/rustc_codegen_llvm/src/llvm_util.rs#L233).
