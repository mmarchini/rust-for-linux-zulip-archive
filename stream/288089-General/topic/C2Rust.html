bjorn3 (Jul 04 2023 at 18:22):

The output of C2Rust will contain all pointer usage unchanged from the C source. This means that it is just as unsafe. Doing a proper rust port requires using higher level concepts like references, Box, Vec, Arc, ForeignOwnable, Pin, ... I don't think it is any easier to turn the output of C2Rust into idiomatic Rust code than it is to write it by hand by looking at the C code.
