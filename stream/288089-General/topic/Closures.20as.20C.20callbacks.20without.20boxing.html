Topic: Closures as C callbacks without boxing

Oliver Mangold (Mar 10 2025 at 09:29):

I am wondering if one can get in work in halfway sane way to pass a small Rust closure to the kind of C callback API we see sometimes in the kernel without having to allocate heap memory.

The kind of API I talk about looks like this:

struct CObjectWithCallback {
    ...
    void (*callback_func)(void *);
    void* user_data;
}

The question is, if the size of the closure is not more than the size of a pointer, could we store the closure somehow in the user_data pointer directly - to avoid the memory allocation - instead of boxing it and setting user_data to Box::into_raw(closure).

It looks to me that would be useful in kernel to do callbacks 'the rust way'.

Unfortunately it seems using mem::transmute for that can result in UB.

We had a discussion about the matter (without finding an elegant solution) here:

https://users.rust-lang.org/t/is-transmuting-a-value-to-a-pointer-and-back-safe/126296

The TLDR is, if the data (here: the closure) contains padding between values, like if it contains an u16 and an u32, llvm treats the padding as unitialized memory. There seems to be no way to make the compiler assume it to be initialized in Rust (without knowing the exact layout of the data, which is impossible for a closure).

I wonder if it would be worth the trouble to make that work anyway somehow, like with some C or assembly helpers. Would that feature be generally useful enough? Thoughts?

Currently I'm doing it like this:

struct Bio {
    ...
    unsafe extern "C" fn callback_wrapper<CB: FnOnce(Bio) + Send + 'static>(bio: *mut bindings::bio) {
        unsafe {
            let cb = KBox::from_raw(bio.read().bi_private as *mut CB);
            let submit_bio = Self(NonNull::new_unchecked(bio));
            KBox::into_inner(cb)(submit_bio);
        }
    }
    pub fn submit_noacct<CB: FnOnce(Bio) + Send + 'static>(self, end_io_callback: CB) -> Result {
        let bio = mem::ManuallyDrop::new(self).as_raw_bio();
        let cb: unsafe extern "C" fn(bio: *mut bindings::bio) = Self::callback_wrapper::<CB>;
        let end_io_callback = KBox::into_raw(KBox::new(end_io_callback, alloc::flags::GFP_NOIO)?);
        unsafe{
            (*bio).bi_end_io = Some(cb);
            (*bio).bi_private = end_io_callback as *const c_void as *mut c_void;
            bindings::submit_bio_noacct(bio);
        }
        Ok(())
    }
    ...
}

As you can see, I allocate a KBox on each call to submit_noacct(). What I would like to do is something like:

    if std::mem::size_of::<T>() <= std::mem::size_of::<*mut c_void>() {
        // UB, depending on what the closure captures
        let end_io_callback = core::mem::transmute<_, *mut c_void>(end_io_callback);
    } else {
        let end_io_callback = KBox::into_raw(KBox::new(end_io_callback, alloc::flags::GFP_NOIO)?);
    }

It probably can be made sound with assembly or C, but correctness will be hard to check (without in-depth knowledge of the compiler). Thus the question if it is worth the trouble.

Oliver Mangold (Mar 17 2025 at 20:27):

Andreas Hindborg schrieb:

I guess some times it will be a fat pointer? A pointer for the function to jump to and a pointer to the captures?

No fat pointer in the regular sense. The function address is represented by the _type_ of the closure, while the capture is represented by it's value. Odd thing, I admit. If you want to have both addresses, you can get the function address by converting the closure to a function pointer and the data address by simply dereferencing the closure. I did that in my code here.

pub fn submit_noacct<CB: FnOnce(Bio) + Send + 'static>(self, end_io_callback: CB) -> Result {
        let bio = mem::ManuallyDrop::new(self).as_raw_bio();
        let cb: unsafe extern "C" fn(bio: *mut bindings::bio) = Self::callback_wrapper::<CB>;
        let end_io_callback = KBox::into_raw(KBox::new(end_io_callback, alloc::flags::GFP_NOIO)?);
        unsafe{
            (*bio).bi_end_io = Some(cb);
            (*bio).bi_private = end_io_callback as *const c_void as *mut c_void;
            bindings::submit_bio_noacct(bio);
        }
        Ok(())
    }
