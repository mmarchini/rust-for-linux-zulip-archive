The thing with the init section is that you cannot use it after initialization. What you need is different, and to my knowledge there is no such feature upstream. You can absolutely do it, as Boqun says. Make a section similar to the init section, but conditionally free it later. It probably requires quite a bit of surgery.

Andreas Hindborg (Jun 26 2024 at 09:57):

Alice Ryhl said:

And so in that case, init would need to load the module from the new initramfs manually?

Yes, this is how most PC and server systems work. You load a kernel and an initial in-memory file system. This file system has the init user space program and some drivers that may be required to mount the root partition. The init program then tries to decide what modules to load, loads them, and tries to mount the root file system.

Linking in all modules for all possible root file systems would make a huge kernel image that would take long to load and initialize. Baking in only the right ones is faster, but requires precise knowledge of the system you are booting. That would not work for something like a generic Ubuntu kernel that needs to boot on a wide selection of systems. The in-memory root file system is the halfway point.
