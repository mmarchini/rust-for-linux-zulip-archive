Miguel Ojeda (Oct 10 2022 at 15:03):

Currently, we still create PRs to use them as a list of things that others are working on. So if you upload your commits there, it would be useful so that others can give an earlier review, maybe base their work on yours, etc.

However, for upstreaming the code to the kernel, a lot of the code that will go into the kernel crate should be sent to the relevant mailing lists (i.e. not just the rust-for-linux one) and reviewed by the relevant kernel maintainers of the C side, as with any other patch. In the ideal case, those maintainers would take responsability for the Rust abstractions.

For starters, we are going to submit some of the core pieces that have been in the rust branch for a while, since they are prerequisites for other things. But of course, feel free to submit things independently if you think they are ready and the prerequisites are there.

As a first step, I would suggest finding out which mailing lists / people should the patches be sent to (using scripts/get_maintainer.pl, checking the git blame, the MAINTAINERS file, etc.).
