Hey! So this isn't super high priority and I likely won't get to it until after rebasing the KMS stuff finally, but after applying Danilo's DeviceContext follow-up work (specifically, the patch which adds T::ParentDevice to drm::Driver) to a local copy of drm-rust-next for another patch series of mine, I ended up with a rather silly pattern that seems to tell me we probably should streamline this:

        if dev.as_raw() != self.dev().as_ref().as_ref().as_raw() {
            return Err(EINVAL);
        }

It seems like it's probably not a bad idea for us to start adding PartialEq to so we don't need as_ref() or as_raw() for doing equality checks

Danilo Krummrich (Jun 27 2026 at 20:30):

You could slightly improve it by writing it as

let parent = self.dev().as_ref();
if dev.as_raw() != parent.as_ref().as_raw() {
    return Err(EINVAL);
}

But implementing PartialEq is a good idea regardless , I remember I saw a patch on the list for this a while ago [1].

Maybe you can check with @Matthew Maurer to pick it up; we could theoretically change his patch to a generic impl for T: AsRef<Device<Ctx>> since Device has an AsRef to itself:

impl<Ctx: DeviceContext> AsRef<Device<Ctx>> for Device<Ctx> {
    #[inline]
    fn as_ref(&self) -> &Device<Ctx> {
        self
    }
}

However, this is dangerous since there's no semantic guarantee about AsRef, so it should rather be a separate trait or limited to just device::Device.
