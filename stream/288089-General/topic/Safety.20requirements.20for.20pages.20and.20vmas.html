Topic: Safety requirements for pages and vmas

Alice Ryhl (Jul 17 2023 at 09:52):

I'd like some help with figuring out the safety requirements for pages and vmas. We can create a page with alloc_pages and free it with __free_page. Additionally, we can map it into a vma using vm_insert_page.

So what are the safety requirements here? What happens if I call __free_page while its mapped in a vma? On the one hand, there seems to be a refcount on the pages. On the other hand, binder never does this. Instead it:

  1. In process cleanup, the vma is already freed, so here it's reasonable that calling __free_page is ok.
  2. In the vma shrinker, it calls zap_page_range before calling __free_page. However, this is part of a memory shrinker operation, and if the vma has a refcount on the page, we want to release that refcount too.

So the above two observations do not really reveal whether there's a refcount that makes __free_page ok while it is mapped with a vma.
