Amélie Gonzalez (Mar 24 2025 at 16:21):

My uninformed opinion: Upsizing everything to usize could have some bad implications if you're doing gcd on a 64-bit integer on a 32-bit architecture. As someone who writes Rust regularly, it always feels kind of, idk, "gross".
On one hand, it's probably less of maintenance pain to wrap gdc() like everything else. On the other, there'll have to be a rust/math.rs file anyways so we could just re-code gcd() in it since it saves time crossing the FFI barrier for things that are trivial to implement Rust-side. The only optimized bit in the C part is if CONFIG_CPU_NO_EFFICIENT_FFS is disabled, and it uses the binary algorithm. Otherwise, it's the even/odd algorithm.

(actually i'm not even sure there's significant time loss from the FFI but i can go check)

While i'm at it, i don't think anybody ever had to write this before, so if anyone needs it at some point, it's free code:

/// Swap two mutable variables that are `Copy`
macro_rules! swap {
    ($a:ident, $b:ident) => {
        let _tmp = $a;
        $a = $b;
        $b = _tmp;
    };
}
