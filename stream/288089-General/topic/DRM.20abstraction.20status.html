To understand the current state of the rust vgem and drm subsystem rust abstraction, I cherry-picked the relevant commits to the latest rust-next branch (Linux 6.6 RC1), and here are the results.

Some commits have been modified so that they can be built on a per-commit basis.
I cherry-picked the order of the commits as much as possible to match the order of the dependent commits so that the build doesn't break.

I shamelessly adopted vgem rust (Maíra Canal's implementation, Linux 6.2.0-rc4 base) commits and Ashahi Linux's DRM driver implementation (Linux 6.4.0 base)'s necessary and related rust layer commits as Monkey do.

  • Ashahi Linux
    - repo: https://github.com/AsahiLinux/linux.git
    - branch: gpu/rust-wip

  • VGEM Rust
    - repo: git@github.com:mairacanal/linux.git
    - branch: vgem/wip-dma

  • Worked repo/branch
    - https://github.com/elongbug/linux/tree/rust-next-drm-wip-vgem

When I run VGEM-related tests using the latest IGT, the results of the dmabuf_sync_file, vgem_basic, and vgem_slow tests are the same as Maíra Canal's VGEM test results. (All passes except kernel crash on vgem_basic-unload. ref: https://mairacanal.github.io/rust-for-vgem/ )

And I have a few questions as I go along, so I leave open questions.
