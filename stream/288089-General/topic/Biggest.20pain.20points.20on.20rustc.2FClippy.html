as soon as we need to start a new project altogether

Ah do you mean that it should be possible to write lints as plugins to rustc, rather than writing a new tool?

In that case, this probably won't happen. Compiler plugins were deprecated a long time ago and are not likely to come back.

So writing a new tool to implement lints will probably always be the case. The only question would be how easy it is to write such a tool / how much boilerplate is required.

klint is already the right approach here. I see some general improvements for it. If you want, I can submit a PR towards klint. E.g. using #![register_tool(klint)], so you can namespace those lints and control them with #[allow(klint::lint_name)]. Ah wait this has to be done in the crate that uses that tool.
