Topic: Proc-macro specific flags?

Matthew Maurer (Oct 16 2024 at 22:47):

In Android, we've got a weird situation where we want the host programs produced by kbuild to be linked against musl (for reasons outside my control). However, the toolchain we use is linked against glibc. This means that:

  • proc-macro crates must be linked against glibc, so that they can be loaded by the compiler
  • Other host rlib/bin crates must be linked against musl

I'm looking for a vibe check on a new env variable called HOSTRUSTTOOLCHAINLDFLAGS before we send to the list. It is intended to add a separate flag that can override HOSTLDFLAGS for proc-macro builds in order to use a different libc for the proc-macros. Preferred flag names? Different ways to inject that would be better?

cc @Alice Ryhl
