Topic: pin_init struct initialization guaranteed order?

Philipp Stanner (Jun 24 2026 at 10:42):

If you initialize a struct with pin_init like so

try_pin_init!( Foo {
  a <- func_one(),
  b <- func_two(),
}

Are you then guaranteed that once you do a KBox::pin_init() on that closure, a and b will execute in order?

Miguel Ojeda (Jun 24 2026 at 10:52):

From https://rust.docs.kernel.org/pin_init/macro.pin_init.html

The fields are initialized in the order that they appear in the initializer. So it is possible to read already initialized fields using raw pointers.

Miguel Ojeda (Jun 24 2026 at 10:54):

There is also the _ blocks syntax that wouldn't make much sense otherwise.
