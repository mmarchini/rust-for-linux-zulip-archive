Rust for Linux: v6.12 - What's new

Note: This is based on the Rust for Linux -next branch. Does anybody have an overview what goes into v6.12 via other trees?

Applied Micro QT2025 PHY driver

This is supposed to be the first functional Rust driver in tree, so no "reference" or "null" driver. It has an out of tree C reference driver, but nothing in tree.

Split up helpers.c

So far the helper functions for all subsystems have been in one file - helpers.c. That led to merge and rebase conflicts, often. As this one file was touched and modified from various parties. To improve this, the content of helpers.c was moved to rust/helpers/ and there each sub component got its own individual file. And, additionally, EXPORT_SYMBOL_GPL() is not required any more. It is auto generated, now.

If you need to rebase some non-merged patches or your out of tree work: Instead of touching helper.c you need to modify your changes and move the helper changes to rust/helpers/. And drop EXPORT_SYMBOL_GPL(). Hopefully that needs to be done only once, expecting less rebase effort in the future with this new layout. And if you add something new please ensure that rust/helpers/helpers.c is kept alphabetically ordered.

Documentation: Add a code documentation section and link to rust.docs.kernel.org

The documentation gets an additional Code Documentation section which links to the newly introduced https://rust.docs.kernel.org & friends.

Add ARef::into_raw

Add a method for ARef that is analogous to Arc::into_raw. It is the inverse operation of ARef::from_raw, and allows you to convert the ARef back into a raw pointer while retaining ownership of the refcount.
