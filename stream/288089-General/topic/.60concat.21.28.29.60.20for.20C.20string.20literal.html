Topic: `concat!()` for C string literal

Boqun Feng (Jul 20 2025 at 16:01):

Context: https://lore.kernel.org/rust-for-linux/aF7lVYl7p33kByoK@tardis.local/

We currently have an optional_name!() macro which either uses the input or the "<file name>:<line number>" as the name. It's primarily used for generating lock class names for LOCKDEP. Now in the workqueue case, we want to use optional_name!() to generate one name and then another name with a "_timer" suffix added. What we could do is extending optional_name!() with an optional suffix parameter. But I wonder, would it be better if we support concat!() a C string literal with a string literal? Like

concat!(c"abc", "def"); // <- generate a c"abcdef"

or only support concat!() within C string literals:

concat!(c"abc", c"def"); // <- generate a c"abcdef"

Thoughts?
