Boqun Feng (Mar 29 2025 at 22:54):

Also notice that the semantics of completion is not the same as the CondVar code above. The above code may miss wake-ups if complete() happens before wait_for_completion:

complete();
wait_for_completion(); // will wait here.

whereas completion API avoids this.

Boqun Feng (Mar 29 2025 at 23:10):

The following is the rough idea of implementing Completion with CondVar, but obvious two problems: 1) completion reuses the wait queue's lock, but here we need another spinlock, 2) completion uses simple wait queue, but CondVar uses normal wait queues. See include/linux/swait.h for the differences.

pub struct Completion {
    done: SpinLock<u32>,
    cond_var: CondVar,
}

impl Completion {
    pub fn wait_for_completion(&self) {
        let mut done = self.done.lock();
        while *done == 0 {
            self.cond_var.wait(&mut done);
        }
        if *done != u32::MAX {
            *done--;
        }
    }

    pub fn complete(&self) {
        let mut done = self.done.lock();

        if *done != u32::MAX {
            *done++;
        }
    }
}
