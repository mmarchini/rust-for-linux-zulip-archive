Topic: Build of v6.8-rc1 fails with undefined symbol __muloti4

Dirk Behme (Jan 25 2024 at 05:39):

In the process of rebasing my local changes from v6.7-rc1 to v6.8-rc1 rust-dev I found that I have to revert

commit 5f3bd90d9 ("buffer: fix grow_buffers() for block size > PAGE_SIZE")

because else cross compiling the kernel from x86 to aarch64 using LLVM=1 fails with

ld.lld: error: undefined symbol: __muloti4
>>> referenced by buffer.c:0 (fs/buffer.c:0)
>>>               fs/buffer.o:(bdev_getblk) in archive vmlinux.a

If I understood correctly the compiler (wrongly?) thinks that it needs to add a 128-bit division here. Reverting above commit "fixes" that issue.

I'm not sure if that is a local tool chain or general problem. I didn't investigate further. But I want to mention it here in case it helps others too :smile:
