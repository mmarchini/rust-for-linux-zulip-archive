Alphyr (Feb 25 2025 at 11:24):

Hi!
I started a branch to change init! and pin_init! to use : instead of <-. This has the advantage of being a more rusty syntax, which may be easier for beginners. It also makes it understandable by rustfmt which is now able to format this code.
On the other hand, it may feel a bit more confusing because more magical.

I couldn't find a reason why this was done with `<-̀ in the first place, so please tell me if changing that was a bad idea.

I pushed a branch here if you want to have a look: https://github.com/a1phyr/linux/tree/improve_init

(Also this makes it possible to change the error type of Init and PinInit traits from a generic to an associated type, which feels more correct but may not be a net positive, see https://github.com/a1phyr/linux/tree/init_assoc_error_type)
