Boqun Feng (Jun 15 2025 at 22:41):

Regarding the pushback from Ingo, I took a look into the patch again.

Since file_with_nul is already merged, I lean towards supporting might_sleep() only when Location::file_with_nul() is supported, in other cases, we can replace the file name with a "<Location::file_with_null() not supported>". This will make the implementation significantly simpler and avoid changing in the future. And since might_sleep() will also trigger a kernel WARN, so the caller information can be found from there.

Please see below:

https://git.kernel.org/pub/scm/linux/kernel/git/boqun/linux.git/log/?h=rust-sched-v2

I will send out the patch once 0day tests pass.
