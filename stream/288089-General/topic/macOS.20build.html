Not exactly. Building the kernel in macOS means building in a “foreign” host. So, it is expected by the user to provide the missing dependencies as well as the environment. They don’t want to deal with a full integration in the build system but it’s okay to modify it and make it “portable” when required. That’s the summary of the feedback in the v1 and v2 series.

It provides the environment as well as the headers.
