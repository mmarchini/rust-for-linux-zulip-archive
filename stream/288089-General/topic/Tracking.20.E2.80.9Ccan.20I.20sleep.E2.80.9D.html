Topic: Tracking “can I sleep”

Demi Obenour (Oct 03 2022 at 02:53):

How can “Can this code sleep?” be tracked at compile-time? It cannot be checked at runtime, and sleeping in (at least) an RCU read-side critical section is unsafe. Naively, this means that the only safe allocation flag is GFP_ATOMIC, but that is a terrible default.

The only approach I can think of is to have some sort of MaySleep ZST passed to every callback that is guaranteed to happen in a sleepable context. Functions that can sleep would take &MaySleep, and entering atomic context would require something like:

pub fn run_rcu_read_side_critical_section<U, T: FnOnce(RCUCriticalSection)-> T>(s: &mut MaySleep) -> T;

The &mut MaySleep would guarantee that no other function that can sleep can be called. One can use a macro to avoid the need for a closure, but this is still rather ugly.

Can one do better than this? @Alex Gaynor @Kees Cook do you have any suggestions?
