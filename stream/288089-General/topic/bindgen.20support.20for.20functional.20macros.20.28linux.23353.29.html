John Baublitz (Jul 27 2022 at 17:04):

so far, what I've seen is that there are two cases with function-like macros. there's the definition which bindgen doesn't need to generate anything for and then there's the expansion of the function-like macros which may be used in constant-like macro expansions and those do need to be evaluated. I've played around with the clang API a bit and I'm not seeing a straightforward way to evaluate the expansion of a macro given that the only thing exposed by clang is the macro name, not the expansion. I have some basic ideas for how we could approach this but I think it involves using using a standalone c preprocessor to get the expansions since the C API for Clang does not seem sufficient. any thoughts so far?
