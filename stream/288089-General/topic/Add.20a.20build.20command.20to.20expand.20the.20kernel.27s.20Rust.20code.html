Topic: Add a build command to expand the kernel's Rust code

JCKeep (Apr 18 2025 at 14:38):

Currently, Rust for Linux doesn’t have a rust_expand command, so if I want to see the expanded
Rust code, it’s a bit of a hassle — I have to run rustc manually with all the right flags.

For example, if I want to see the expanded Rust code for rust/kernel, I have to run this long command:

OBJTREE=/home/jckeep/C/linux rustc +nightly --edition=2021 -Zunpretty=expanded rust/kernel/lib.rs \
    -Cpanic=abort -Cembed-bitcode=n -Clto=n -Cforce-unwind-tables=n -Ccodegen-units=1 \
    -Csymbol-mangling-version=v0 -Crelocation-model=static -Zfunction-sections=n \
    -Ctarget-feature="-neon" -Cforce-unwind-tables=n -Zbranch-protection=bti,pac-ret \
    -Copt-level=2 -Cdebug-assertions=y -Coverflow-checks=y -Cforce-frame-pointers=y -Cdebuginfo=1 \
    --crate-name kernel --extern ffi --extern build_error --extern macros --extern bindings --extern uapi \
    -L./rust --target aarch64-unknown-none @./include/generated/rustc_cfg  --crate-type=rlib

That’s way too much. Maybe we could add a simple command to expand the kernel’s Rust code more easily. :thinking:
