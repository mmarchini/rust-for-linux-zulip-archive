For this cargo supports both path dependencies (using the path key when specifying a dependency) and vendoring (using cargo vendor). Mozilla also recently released cargo-vet for help with auditing dependencies. Whether that will be considered enough is another question. It should also be possible to replace the crates-io registry with an empty registry to prevent having any dependencies from crates.io.
