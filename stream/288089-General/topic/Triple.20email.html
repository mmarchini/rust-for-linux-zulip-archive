Martin Rodriguez Reboredo (Aug 05 2023 at 12:50):

I've sent the email using the following command under the linux tree.

git send-email --to="rust-for-linux@vger.kernel.org" \
    --cc-cmd="./scripts/get_maintainer.pl --no-rolestats v3-0001-scripts*.patch" \
    v3-0001-scripts-generate_rust_analyzer-provide-cfg-s-for-.patch

Indeed, it's very powerful, you can also test it using the --dry-run flag with it. And certainly, the reason of this thread is because -n is a common shortcut of --dry-run in git, take for instance git-clean, although if you pass it to git-send-email it seems that it'll be ignored and things won't get run dry. Perhaps I should do a PR for git.

You can put config into .git/config so that you don't have to repeat. E.g. I use this config:

[sendemail]
tocmd = scripts/get_maintainer.pl --norolestats --nol
cccmd = scripts/get_maintainer.pl --norolestats --nom --nor
