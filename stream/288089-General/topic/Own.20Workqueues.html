John Hubbard (Nov 12 2025 at 19:27):

I remember running into the need for specific workqueues, back when I was working on unified memory (page faulting for GPU and system memory, and migrating pages back and forth between GPU and CPU memory).

The system workqueue couldn't be used because it would have lead to deadlocks. For complex drivers, this is going to be routinely required.

I expect that Nova will eventually need it, although I might be wrong, because we'll be layering on top of several subsystems that already work well (HMM, DRM), so we'll see.
