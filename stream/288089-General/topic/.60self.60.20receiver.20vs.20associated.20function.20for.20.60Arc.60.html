Topic: `self` receiver vs associated function for `Arc`

Benno Lossin (Sep 23 2024 at 09:23):

The convention is to use associated functions, since they don't shadow the functions on the values contained inside. Do we want to use that convention for Arc? Current functions that don't follow that rule:

  • into_raw
  • into_unique_or_drop
  • assume_init

Additionally, ForeignOwnable::into_foreign has a self receiver.

For example into_unique_or_drop one could argue that there never will be a collision. But it might still happen (and then people might be confused why it's not calling the right function) and if another function that should be associated is added, people might think that using self is fine, since other functions already use it.

Any thoughts?
