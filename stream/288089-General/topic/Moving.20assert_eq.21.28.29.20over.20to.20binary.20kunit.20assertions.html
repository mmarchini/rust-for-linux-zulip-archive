Topic: Moving assert_eq!() over to binary kunit assertions

Lyude (Oct 13 2025 at 16:35):

Hey! I don't need any help with this, mainly just posting this to make sure no one else was working on it. I noticed the other day that currently assert_eq!() in kunit tests actually uses unary assertions, which means if have an assert like this fail:

assert_eq!(some_num, 5)

The test doesn't actually print the value of some_num. An example from a different line of code that motivated me to fix this in the first place:

[19:12:30] # rust_doctest_kernel_iosys_map_rs_0.location: rust/kernel/iosys_map.rs:117
[19:12:30] # rust_doctest_kernel_iosys_map_rs_0: ASSERTION FAILED at rust/kernel/iosys_map.rs:126
[19:12:30] Expected *map.as_slice(0, 3)? == [0; 3] to be true, but is false
[19:12:30] [FAILED] rust_doctest_kernel_iosys_map_rs_0

Anyway - mostly just writing this in case someone else is already working on this, otherwise I'll try to send a fix to the mailing list today or tomorrow
