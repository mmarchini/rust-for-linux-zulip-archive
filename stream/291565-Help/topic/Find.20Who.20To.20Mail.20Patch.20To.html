I'm not sure what you mean by "supporters", maybe you mean the maintainers in the M: entries? I don't actually know if there is a hard rule by which to decide if to use TO or CC... But I have seen mixes of both. The most important part is that you send the mail to the list and the people listed in the entry.

I use the following configuration in git:

[sendemail]
    tocmd = scripts/get_maintainer.pl --norolestats --nogit-fallback --nol
    cccmd = scripts/get_maintainer.pl --norolestats --nogit-fallback --nom --nor

Then it automatically picks the correct maintainers for the patch (but no guarantees of course :) when sending via git send-email. (it puts reviewers and maintainers in TO and the list in CC)

Have you also seen https://docs.kernel.org/process/submitting-patches.html ? It contains a lot of other useful information.
