What is the purpose of rust_helper_fsleep()? At first I thought it was because fsleep() is an inline function, so technically it doesn't exist in memory, but then I noticed that rust_helper_fsleep() is never actually called, and the Rust function calls bindings::fsleep() anyway.

Alice Ryhl (Apr 28 2025 at 19:46):

bindings::fsleep() will reference the helper if no non-helper version exists

Timur Tabi (Apr 28 2025 at 19:47):

How, exactly? This must be some feature of Rust I don't know about. And what is a "non-helper version"?

Alice Ryhl (Apr 28 2025 at 19:50):

Bindgen runs on both rust/bindings/bindings_helper.rs and on the files in rust/helpers. A regex is used on bindgen output to rename the ones in rust/helper to drop the rust_helper_ prefix. The bindgen crate imports both bindgen output files using the right glob imports to prefer one over the other.

Timur Tabi (Apr 28 2025 at 19:51):

Alice Ryhl (Apr 28 2025 at 19:51):

Alice Ryhl (Apr 28 2025 at 19:52):

Dropping the rust_helper_ prefix isn't done by bindgen. It's a literal regex we run on the file afterwards. See here.
