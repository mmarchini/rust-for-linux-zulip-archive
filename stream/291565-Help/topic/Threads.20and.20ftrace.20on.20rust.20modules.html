Boqun Feng (Oct 27 2024 at 18:20):

For thread, even in C, creating a thread is not encouraged as kernel threads are important resources and justification is needed to use a kernel thread. The general advice is to use something like workqueue. So please look into using workqueue for your usage.

But I did have a PR back in the old time. And just rebase on the latest rust-next: https://github.com/fbq/linux/tree/dev/rust/thread

For ftrace, it's currently not supported, but it shouldn't be that hard (famous last words), IIUC, you will just need to make sure Rust functions work with ftrace_modify_all_code() (maybe additional calling convention need to be handled as well?)
