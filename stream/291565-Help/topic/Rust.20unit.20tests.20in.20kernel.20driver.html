Yeah, this looks to be a problem with the kunitconfig: kunit.py definitely shouldn't always do a full rebuild. It looks like the kunitconfig file you're using is a full .config, when it really should only be a list of the few (non-default) options actually required for the test (and there dependencies).

It's probably worth taking a look at the KUnit docs, as well as @José Expósito's KUnit/Rust doc PR. As-is, this seems to work for me as a cut down version of your kunitconfig:

CONFIG_RUST=y
CONFIG_KUNIT=y
CONFIG_ANDROID_BINDER_IPC_RUST=y
CONFIG_BINDER_TEST=y
CONFIG_RUST_KERNEL_KUNIT_TEST=y

Which I can then run with:

./tools/testing/kunit/kunit.py run --kunitconfig kunitconfig_rust_binder --make_options LLVM=1 --arch x86_64

Which completes pretty quickly as a nothing-changed incremental build:
