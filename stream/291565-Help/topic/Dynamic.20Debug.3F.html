Andrew Ballance (May 23 2025 at 11:47):

I got nerd sniped on this.
I have a somewhat working demo. It is held together by duct tape and hope.
link

couple of things to note:
the c version uses the __func__ macro to get the name of the current function. there is no equivalent in rust.
It looks like there was a proposal to work on it but it fell through.
https://github.com/rust-lang/rfcs/pull/2818

the c version uses the KBUILD_MODNAME macro to set the modname. it doesn't look like there is currently a rust equivalent. we could pass KBUILD_MODNAME to rustc as an enviroment varible to set the modname of the debug info. and we probably should make getting the modname its own dedicated rust macro.

Trevor Gross (May 23 2025 at 11:53):
