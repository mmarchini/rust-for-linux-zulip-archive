Amélie Gonzalez (Nov 08 2023 at 09:08):

As I understand it, work in the rust branch was primarily experimental and trying to see how the project could be built and what could be done with it. It has since been deprecated in favour of a proper development process tied to the rest of the Linux kernel through rust-next (for upstreaming) and rust-dev (for new features to be eventually upstreamed). Everything that was implemented in rust has not been re-made and reviewed yet to appear in either, and I think FileOperations and miscdev::Registration are among those. That makes my own little toy chardev module (which I assume was kind of following the same old tutorials you're using) unusable with it as well.

That being said, there is an ongoing patch set trying to add some basic file system support, and I haven't combed through it thoroughly so it's either a first step in the direction you want, or adds something similar to what you are trying to use.

On another note if you just want to tinker and not care immediately about merging into mainline (trust me it's not gonna happen anytime soon; not to demoralize you or anything but it's a whole process), you can just… straight up get the code from rust that you are missing, slap it over rust-dev, and start hacking (I did that for some netdev and it mostly works).
