Hongyu Li (Feb 24 2022 at 15:09):

Thanks for your help sincerely! After some thinking and investigation, I convince myself finally. But I don't know whether the reason is right.

It seems most of the functions related to the struct file in the Linux kernel use a pointer to the struct file as the argument or return value. The reason for this is to decrease the memory size of the parameters. Therefore, in the bindings_generated.rs, the function parameter related to the struct file is "*mut file". Therefore, in the file.rs we use the struct File to wrap the pointer, which helps us from writing duplicated codes.

Hongyu Li (Feb 24 2022 at 15:11):

We don't need to write *mut bindings::file every time. Instead, we just need to write "self.ptr" to represent *mut bindings::file in the impl File block.

Wei Liu (Feb 24 2022 at 15:14):

Well, I would say memory usage concern is secondary in Linux kernel (not specific to Rust code). It doesn't make sense to have multiple copies of the same thing. There should be one canonical copy that holds all the information.

Wei Liu (Feb 24 2022 at 15:17):

Rust for Linux consumers (as in, engineers not working on core abstractions) shouldn't really try to mess with raw pointers. The File abstraction is more about providing safe abstraction for consumers. The reason is not about avoiding code duplication.

Hongyu Li (Feb 24 2022 at 15:28):
