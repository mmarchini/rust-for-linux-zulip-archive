Topic: scripts/generate_rust_analyzer.py fails on NixOS

Onur Özkan (Dec 24 2025 at 11:04):

Just recently I switched my development environment to NixOS and I am trying to make isolated environment for each project I am working on.

For linux, I have this nix shell:

{ pkgs ? import <nixpkgs> {} }:

pkgs.mkShell {
  packages = with pkgs; [
    bison
    elfutils
    flex
    gcc
    glibc.dev
    llvmPackages.clang-unwrapped
    llvmPackages.lld
    llvmPackages.llvm
    openssl.dev
    pkg-config
    python3
    qemu
  ];

  nativeBuildInputs = with pkgs; [
    rustc
    cargo
    rust-bindgen
    rustfmt
    clippy
  ];

  RUST_LIB_SRC = pkgs.rustPlatform.rustLibSrc;

  shellHook = ''
    export RUST_SRC_PATH="$RUST_LIB_SRC"
    export BINDGEN_EXTRA_CLANG_ARGS="-Wno-unused-command-line-argument"
    alias make='make HOSTCC=gcc'
  '';
}

but the scripts/generate_rust_analyzer.py script fails on the sysroot sanity assertion here. Anyone else hitting this issue on NixOS or is it only me? FWIW other than make LLVM=1 rust-analyzer, I can build the kernel just fine.
