Topic: Private bounds warning

Philipp Stanner (May 26 2026 at 12:54):

I've got a private trait that I implement on data that ultimately comes from the API user, to enforce compile-time checks on whether that data either implements Copy or is wrapped in another type.

One of my structs containing such data is public, but the trait when passing the data on creation is private. This fires this warning:

note: but trait `DriverFenceAllowedData` is only usable at visibility `pub(self)`
   --> rust/kernel/dma_buf/dma_fence.rs:488:1
    |
488 | trait DriverFenceAllowedData {}
    | ^^^^^^^^^^^^^^^^^^^^^^^^^^^^
    = note: `#[warn(private_bounds)]` on by default

I would have liked to switch the warning of, but allow has no effect

/// A trait to enforce that all data in a [`DriverFence`] either does not need
/// drop, or lives in a [`RcuBox`].
#[allow(private_bounds)]
trait DriverFenceAllowedData {}

impl<F: Copy> DriverFenceAllowedData for F {}
impl<F: Send> DriverFenceAllowedData for RcuBox<F> {}

Manos Pitsidianakis (May 26 2026 at 12:59):

Consider making the trait public, and use the Sealed pattern. (search for trait Sealed in rust/kernel to see examples)

Philipp Stanner (May 26 2026 at 13:02):

Why's it even illegal to have the trait be private?

Manos Pitsidianakis (May 26 2026 at 13:02):

Because if it's a bound, users of your API cannot use it as a bound themselves.

Manos Pitsidianakis (May 26 2026 at 13:11):

If your API is:

trait Foo;

impl Foo for bool {}
impl Foo for u8 {}

pub fn accept_foo<F: Foo>(foo: F) {
  ...
}

Then the API user can do stuff like:

accept_foo(true);
accept_foo(1_u8);

But if they want to do something like:

fn validate_foo<F: ????>(is_valid: bool, foo: F) {
  if is_valid {
    accept_foo(foo);
  }
}

Then they need to access the trait bound to import it.

Manos Pitsidianakis (May 26 2026 at 13:12):

The Sealed pattern allows you to prevent people from implementing your public trait and let them refer to it if they want to.

Gary Guo (May 26 2026 at 14:06):

You're marking #[allow] in the wrong place, it should be where you use the trait, not on the trait.
