Miguel Ojeda (Jan 26 2026 at 12:00):

And for our "good first issue" list, yeah, we add more from time to time as time allows. But you can think about similar issues you could tackle as a first patch by looking into old ones and going around the kernel Rust source code to see if you notice things that could be improved.

Benno Lossin (Jan 26 2026 at 12:00):

If you want to work on pin-init, there is this issue: https://github.com/Rust-for-Linux/pin-init/issues/85 that I want to implement for the next cycle. If you're experienced with rust proc macros, then that could be something you could work on. I already have a partially working solution in the dev/tuple-structs branch

Benno Lossin (Jan 26 2026 at 12:01):

if you're not too experienced with Rust, I'd not recommend working on pin-init, as it's quite complex. It also isn't interacting with the kernel, so you'd be writing just normal #![no_std] Rust code.

Miguel Ojeda (Jan 26 2026 at 12:02):

Finally, please note that there are other ways of contributing, not just adding new code! For instance, reviewing code and testing is something we will need more and more, and some people do like to do that, so I always mention it in case it is not obvious. It also gives credit in the Git log messages. The same applies to testing too.
