Ariel Miculas (Oct 07 2024 at 17:56):

I guess you need to use pkgsStatic.busybox, figure out where the rootfs gets copied, then go from there. I can see that pkgStatic.busybox installed the following on my system:

$ ls -la /nix/store/hcdjs6yjqmw5a5y4gilrlvq855m7igyv-busybox-static-x86_64-unknown-linux-musl-1.36.1/
total 784
dr-xr-xr-x    3 root     root          4096 Jan  1  1970 .
drwxrwxr-t 1151 root     nixbld      778240 Oct  7 20:37 ..
dr-xr-xr-x    2 root     root         12288 Jan  1  1970 bin
-r-xr-xr-x    1 root     root          2274 Jan  1  1970 default.script
lrwxrwxrwx    1 root     root            11 Jan  1  1970 linuxrc -> bin/busybox
lrwxrwxrwx    1 root     root             3 Jan  1  1970 sbin -> bin

where the bin has all the busybox binaries. So if you can get this path, you can add the init script and then create your rootfs.

Ariel Miculas (Oct 07 2024 at 23:26):

Pasha Fistanto said:

Hmm that means compiling the busybox manually is not recommended on nixos?

Well I believe you would do it like this: https://gist.github.com/ariel-miculas/41d5e62d94f9927e4c21e6dc1019277f
Then run the following steps:

$ nix develop --extra-experimental-features nix-command --extra-experimental-features flakes
amiculas@amiculas-l-PF3FCGJH:~/work/kernel-devel$ which busybox
/nix/store/vawdvyxri5hx2mpgwf9iixdrcb4ggnsj-busybox-1.36.1/bin/busybox
amiculas@amiculas-l-PF3FCGJH:~/work/kernel-devel$ cd /nix/store/vawdvyxri5hx2mpgwf9iixdrcb4ggnsj-busybox-1.36.1/
amiculas@amiculas-l-PF3FCGJH:/nix/store/vawdvyxri5hx2mpgwf9iixdrcb4ggnsj-busybox-1.36.1$ find . | cpio -H newc -o | gzip > ~/work/kernel-devel/ramdisk.img
5308 blocks
amiculas@amiculas-l-PF3FCGJH:/nix/store/vawdvyxri5hx2mpgwf9iixdrcb4ggnsj-busybox-1.36.1$ cd -
/home/amiculas/work/kernel-devel
amiculas@amiculas-l-PF3FCGJH:~/work/kernel-devel$ qemu-system-x86_64 -accel kvm -cpu host -m 4G -initrd ramdisk.img -kernel ~/work/linux/arch/x86/boot/bzImage -nographic -append 'console=ttyS0 nokaslr debug rdinit=/init' -nic user,model=rtl8139 -no-reboot

Here I'm assuming you have already built the kernel with qemu-busybox-min.config and rust enabled.
But there are probably easier ways, as Gary mentioned.
