Topic: ✔ How to send a PR relaying on other PRs?

Hongyu Li (Aug 24 2022 at 01:51):

Hi. I plan to implement the virtio-net driver in rust. Many APIs used in virtio-net are still not supported in rust-for-linux. If I put the API wrapping code and virtio-net driver code in a PR, it will be too huge to review. Thus, I plan to split it into servals PRs and open an issue to track their states.

However, the virtio-net driver code relays on the API wrapping code, I have to wait till all the APIs are merged and then send the virtio-net driver. Is there a way to send the pre-version virtio-net?

Boqun Feng (Aug 24 2022 at 01:58):

You can have two PRs, one based on the other, one includes all the changes for virtio-net, and the other only has the changes for API wrapping.

Boqun Feng (Aug 24 2022 at 02:01):

For example:

  • https://github.com/Rust-for-Linux/linux/pull/798
  • https://github.com/Rust-for-Linux/linux/pull/796

Boqun Feng (Aug 24 2022 at 02:20):

It depends, you can

  • send a new PR based on this one, but that also means you will need to resolve comments on two PRs, and probably rebase the new PR because of the changes in the old.
  • or you can add your new APIs in the existing PR, and probably describe the changes in the "new version" with some comments, the downside is that, since now it's a single PR, you will need to wait for all comments to all the APIs resolved.
