Jesung Yang (May 24 2025 at 15:25):

Environments:

  • linux: Current latest rust-next tree
  • toolchains: stable-aarch64-unknown-linux-gnu
  • rust-analyzer: Visual Studio Code extension, version 0.4.2471
  • All actions are performed inside a Docker container based on rust:1.87.0-bookworm (not sure if this is relevant)

Hi! I've generated rust-project.json using make LLVM=1 rust-analyzer, but rust-analyzer fails with the following error.

2025-05-24T14:56:53.110157159Z ERROR FetchWorkspaceError: rust-analyzer failed to load workspace: Failed to load the project at /root/workspace/linux/rust-project.json: Failed to deserialize json file /root/workspace/linux/rust-project.json: Invalid cfg ("--edition=2024"), value should be in quotes at line 7 column 13

Here is a snippet from the generated rust-project.json.

{
    "crates": [
        {
            "cfg": [
                "no_fp_fmt_parse",
                "--edition=2024"
            ],
            "deps": [],
            "display_name": "core",
            "edition": "2021",
            "env": {
                "RUST_MODFILE": "This is only for rust-analyzer"
            },
            "is_proc_macro": false,
            "is_workspace_member": false,
            "root_module": "/usr/local/rustup/toolchains/stable-aarch64-unknown-linux-gnu/lib/rustlib/src/rust/library/core/src/lib.rs"
        },
...

If I remove "--edition=2024" from the cfg array, it works fine. But if feels like a workaround rather than a proper solution. Any help or guidance would be appreciated!

UPDATE:
I think this problem is related to commit fffcd7ce47888c9f6d90f9fe803d0fc288b35fb2 (rust: compile libcore with edition 2024 for 1.87+), since it changes rust/Makefile:

+ ifeq ($(call rustc-min-version,108700),y)
+ core-cfgs += --edition=2024
+ else
+ core-cfgs += --edition=2021
+ endif
