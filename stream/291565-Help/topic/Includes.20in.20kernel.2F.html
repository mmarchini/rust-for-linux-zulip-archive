Miguel Ojeda (Feb 14 2023 at 17:02):

Yeah, for the moment I would do that. Note that for private APIs, we will need to be very clear to upstream maintainers that we will use them on Rust's side.

Miguel Ojeda (Feb 14 2023 at 17:03):

Eventually, when the kernel crate is split, we will want to limit the access to such private C APIs to particular crates, i.e. it would really be a case where the Rust code is considered part of the same privacy boundary as the C side, even if written in a different language.
