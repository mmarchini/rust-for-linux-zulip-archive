Benno Lossin (Aug 24 2024 at 15:39):

The networking abstractions that you used are in the old rust branch. As Miguel already said, those are from when RfL was still out of tree.

New kernel code can only be accepted if there exists a user (ie a driver). Nobody has yet written a network driver in Rust, so no abstractions exist.

We are aware that this is a bit of a chicken and egg problem (no driver -> no abstractions -> hard to write a driver...).

You said that you wanted to try it as a learning experience, so I guess you don't want the driver to be upstreamed, right?
