Miguel Ojeda (Sep 04 2022 at 12:02):

is that some of the config flags have value "n" ( !MODVERSIONS [=n] , !RANDSTRUCT [=n], !DEBUG_INFO_BTF [=n]) which eventually makes RUST config also to "n"?

Almost! The problem is !GCC_PLUGINS [=y], i.e. the result is n. So if you disabled GCC plugin support, you should see the option appear.

Having said that, Wedson's suggestion (LLVM=1) is what you should do unless you have a good reason not to, because GCC support is just best-effort at this point in time :)
