Andreas Hindborg (May 05 2025 at 07:56):

@Miguel Ojeda @Gary Guo I figured out what was wrong at my end. It was two issues on op of each other.

1) If you build arch/arm64/lib/xor-neon.o directly with make ... arch/arm64/lib/xor-neon.o, flags are not set correctly. But building the file as part of a full kernel build works fine. This was my test case and thus I was blind as to what worked. Tooling from kernel.org work fine in my setup when building the full kernel.

2) The default clang in nix is built with gcc and uses gcc resources at runtime. This includes some include headers and I believe also parts of libexec. So I was getting arm_neon.h from gcc, which did not work. Using instead pkgs.llvmPackages.clangUseLLVM, the cross build works fine.
