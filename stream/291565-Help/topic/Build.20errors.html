So that code is in the Rust panic handler, and currently we map it to a kernel panic.

Rust would not allow you to compile it if you both remove the BUG and the loop -- the function would be returning but it does not return.

I guess it won't hurt to have it documented as it's the commit which introduced the problem in bindgen in the first place. It worked prior to LLVM-16, I guess that's my rationale for adding it.

Demi Obenour (Sep 12 2023 at 20:07):

Thanks!

In general, the kernel will not build with newer Rust versions, due to the different unstable features (and the alloc fork). We have some notes about it in https://rust-for-linux.com/rust-version-policy and https://docs.kernel.org/process/changes.html#rust-optional.

This is going to be a problem for distributions, which cannot use rustup as part of the build.

Boqun Feng (Sep 12 2023 at 20:12):

I think whether rustup can be used is irrelevant here? As long as distro can build the kernel with the required rustc version.

Miguel Ojeda (Sep 12 2023 at 22:19):

Yeah, rustup is not needed to build the kernel.

Some distributions, when I checked with them, told me they would be happy to provide a package (or support in some other way) to compile the kernel with the latest Rust release, assuming they need it.

Ubuntu, for instance, offers experimental support for it, by providing a package with the required Rust toolchain.
