I try to track toolchain related stuff in our usual lists from #2. We never had the compiler hanging from what I recall, but we have definitely hit e.g. ICEs in the past, and issues generally get fixed when we report them upstream.

I can reproduce the hang -- using the new trait solver on kernel gives a hint:

This hangs it:

trait Bound {}

trait Alloc {
    const OPS: u32;
}

trait Driver {
    type Object<Ctx: Bound>: Alloc;
}

fn f<T: Driver>() -> u32 {
    T::Object::OPS
}

which one can easily see in e.g. Compiler Explorer. It may be one of the existing issues they have, but I will fill it anyway.
