Asahi Lina (May 01 2023 at 09:11):

-mindirect-branch-cs-prefix was added in clang 16.0.0. I assume you're building with LLVM=1, and then that option doesn't get filtered from bindgen since it uses the clang options directly, but due to the version mismatch bindgen can't handle it. It looks like it only gets used on x86 with CONFIG_RETPOLINE, so maybe try disabling that? For the other one, make sure you have the patch Miguel mentioned or disable CONFIG_INIT_STACK_ALL_ZERO. For the last one, that's also probably caused by the mismatch, since the kernel decides what -Wno-* arguments to pass based on the clang version, and then that mismatches libclang.

@Miguel Ojeda It looks like LLVM=1 with a major version mismatch between libclang and clang is almost always going to fail given the way the kernel constructs C compiler arguments... I think we should make that diagnostic clearer, maybe adding something like "this will likely cause unknown argument errors". I don't think we can support those mismatches easily if we want to share C compiler options between clang and bindgen...
