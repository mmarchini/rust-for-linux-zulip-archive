Benno Lossin (Nov 03 2023 at 23:27):

Benno Lossin said:

a function from the standard library that also uses this pattern is https://doc.rust-lang.org/std/thread/fn.scope.html

scope allows you to borrow non 'static data. This means that at some point the data borrowed by the threads might no longer exist. For this reason the threads need to terminate before that happens. If we did not have the branded type API, but just a normal rust type, then the following would be possible:

let scope = thread::scope();
let handle = scope.spawn(|| {
     loop {
         // do something with non `'static` data
     }
});
forget(handle);
forget(scope);

Now nobody ensures that the thread actually terminates before the data is no longer alive.
