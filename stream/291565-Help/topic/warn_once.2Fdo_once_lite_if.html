Jens Korinth (Oct 17 2024 at 19:55):

Hi everyone!
Noticed that there is no support yet for WARN_ONCE, so I tested out an implementation that works exactly like the "lite" variant in the kernel using a static bool:

macro_rules! do_once_lite_if {
    ($cond: expr, $e: expr) => {
        #[link_section = ".data.once"]
        static mut __ALREADY_DONE: bool = false;
        if unlikely($cond && unsafe { !__ALREADY_DONE }) {
            unsafe { __ALREADY_DONE = true; }
            $e;
        }
    }
}

macro_rules! warn_once {
    ($($arg:tt)*) => (
        do_once_lite_if!(true, crate::pr_warn!($($arg)*))
    )
}

This seems to work but I was wondering if this kind of unsafe usage is ok? Worst case would be that the do once part is done nproc times instead of once; but that's also true for the C version.

2nd question: How do you replace unlikely? I've read that #[cold] can be used to achieve a similar effect, but I don't really see how?

After the first iteration, the compiler knows that the boolean must be true. Either it started out true, or we just set it to true. And since you're using unsynchronized loads, the compiler knows that other threads can't modify the value in parallel, so there's no way it could have changed. Thus, reading the global must return true, skipping the loop. Same logic applies to iteration 3 and 4 and so on.
