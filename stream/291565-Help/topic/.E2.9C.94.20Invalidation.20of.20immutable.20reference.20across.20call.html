Topic: ✔ Invalidation of immutable reference across call

Andreas Hindborg (Jan 18 2024 at 14:38):

For this program

extern "Rust" { fn foo(p: *const u64); }

pub fn main() -> u64{
    let x: u64 = 5;
    let y = &x;
    let z = y as *const u64;

    unsafe {foo(z)} ;

    *y

the compiler will insert a load though y before returning:

example::main:
        pushq   %rax
        movq    $5, (%rsp)
        movq    %rsp, %rdi
        callq   *foo@GOTPCREL(%rip)
        movq    (%rsp), %rax
        popq    %rcx
        retq

But y is an immutable reference and it is live across the call to foo(). So the compiler should be able to assume that it did not change, and there is no UnsafeCell involved.

Does anyone know where this behavior is documented? I can't find it anywhere.

Andreas Hindborg (Jan 18 2024 at 19:22):

I was able to write a program that shows the behaviour:

extern "C" { fn foo(p: *const u64) -> u64; }

pub extern "C" fn bar(x: &u64) -> u64 {
    let y = *x;
    let z = unsafe { foo(x) };
    z+y+*x
}

example::bar:
        pushq   %rbx
        movq    (%rdi), %rbx
        callq   *foo@GOTPCREL(%rip)
        leaq    (%rax,%rbx,2), %rax
        popq    %rbx
        retq

The instruction leaq (%rax,%rbx,2), %rax adds 2x *x as it was loaded before the call to the return value of the call.
