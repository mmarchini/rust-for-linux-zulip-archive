Dirk Behme (Nov 05 2024 at 11:23):

I use

make x86_64_defconfig LLVM=1; make -j8 LLVM=1

and the result runs with QEMU system emulation

qemu-system-x86_64 -kernel arch/x86_64/boot/bzImage -nographic -append "console=ttyS0"

(qemu-system-x86_64 might be Ubuntu specific?)

The x86_64_defconfig doesn't have CONFIG_RUST enabled, though. So you need to enable that.

Dirk Behme (Nov 06 2024 at 07:26):

wenzhao said:

with the command

qemu-system-x86_64 -kernel arch/x86_64/boot/bzImage

First of all, good that it works for you, now :smile:

The

end Kernel panic - not syncing : VFS : unable to mount root fs on ""  or unknown block (0,0)

is expected as we have given no root file system. So your kernel boots but doesn't find any root file system/ user space. If you just want to test compiled in code or for example running the Rust tests this is sufficient, already. Anyhow, you can add a root file system (image) of your choice with the additional QEMU option -initrd. I found this helpful, but there are many, many root file system image options out there. Choose what you need :smile:
