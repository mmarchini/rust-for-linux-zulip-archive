Alice Ryhl (May 17 2023 at 10:21):

There are three relevant permissions:

  1. Access value mutably from threads other than the one it was created on, with synchronization. (Send gives this.)
  2. Access value immutably from threads other than the one it was created on, with synchronization. (Send and Sync both gives this - you only need one of them to have it.)
  3. Access value immutably from threads other than the one it was created on, without synchronization. (Sync gives this.)

Alice Ryhl (May 17 2023 at 10:33):

No, I mean that the access must be synchronized externally. That is, the access must happen one thread at the time.

Alice Ryhl (May 17 2023 at 12:02):

Ariel Miculas said:

or does it break rule 1 because it doesn't use synchronization when updating its reference counter?

To expand on this, I don't think its meaningful to say that the type itself can break these rules. From the perspective of external code, they are rules that the external code must follow (that is, it makes sense to talk about whether external code breaks or follows them.) However, from the perspective of the implementation of the type, the rules are not rules, but permissions that the implementation gives to unknown user code. You can talk about whether it is incorrect to give out that permission, but you can't really talk about breaking a permission per se when you are the one who gives out the permission.
