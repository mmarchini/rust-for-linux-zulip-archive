Topic: ✔ why driver needs a "Pin<Box<Registration<...>>>"?

Hongyu Li (Apr 15 2022 at 10:06):

Hi. I recently need to write a driver capable of creating some chrdevs. I find I need to add all the registration items to the struct implementing kernel::Module trait if I want to see these chrdevs in the /proc/devices. The name of the registration items has no influence, but the content matters. It needs to be Pin<Box<chrdev::Registration<N>>>.

The struct should be

struct RustChrdev {
    _dev: Pin<Box<chrdev::Registration<N>>>,
}

rather than

struct RustChrdev;

I read the source code of the macro in the function module. The driver struct is matched as __MOD as follows.

fn __init() -> kernel::c_types::c_int {{
    match <{type_} as kernel::Module>::init(kernel::c_str!(\"{name}\"), &THIS_MODULE) {{
        Ok(m) => {{
            unsafe {{
                __MOD = Some(m);
            }}
            return 0;
        }}
        Err(e) => {{
            return e.to_kernel_errno();
        }}
    }}
}}

I don't find any other routines about __MOD instead of drop. I don't understand why we need to put the Registration in the driver struct. The alloc_chrdev_region and cdev_add are already called, so the chrdev should be created. Thanks for your help!

Alex Gaynor (Apr 15 2022 at 11:57):

__exit will drop __MOD, which in turn recursively drops things. chrdev::Registration has a Drop implementation which de-registers the device.

Hongyu Li (Apr 15 2022 at 15:00):

Alex Gaynor (Apr 15 2022 at 15:13):

If you don't put it in the struct then chrdev_reg is dropped as soon as init returns, and dropping that struct deregisters the value.

Wei Liu (Apr 15 2022 at 15:13):

Your chrdev_reg is dropped after init returns, no?

I have some questions further upon Pin<Box<T>>. Why must Registration be pinned? Could it just be on the stack without allocation?

Hongyu Li (May 22 2022 at 09:31):

The Registration has a member of the C struct cdev, with the list-head inside. Therefore the struct Registration is self-referential. This requires Registration to be pinned. Wedson explained in detail why using Pin in this video.
