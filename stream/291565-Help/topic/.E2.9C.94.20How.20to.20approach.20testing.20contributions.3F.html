Miguel Ojeda (Apr 23 2026 at 12:51):

I think both are good -- it is of course good to test complex logic since bugs can hide there easily, but it is also fine testing small things (e.g. in many "trivial" cases, this is covered by the doctests already, so adding extra unit tests may not add much value).

In general, we want to encourage developers to write tests (wherever reasonable). It is usually the case it would be nice to have more tests, rather than less, i.e. having "too many tests" it is not something that comes often as an issue so far (it would be a nice issue to have, in a way :).

By the way, another kind of test (apart from unit tests and doctests) are adding static/debug/runtime/... asserts wherever possible. For instance, adding debug_assert!s is a good thing if we know something must be true, or that some code should never be reached unless there is a major bug somewhere, etc. One particular one that I would like to see developers using more is the new unsafe_precondition_assert!. So patches that add some of those would be nice (there are some existing debug_assert!s that can be renamed -- actually, let me create a good first issue for that: https://github.com/Rust-for-Linux/linux/issues/1232).

I hope that helps, and thanks for working on testing!
