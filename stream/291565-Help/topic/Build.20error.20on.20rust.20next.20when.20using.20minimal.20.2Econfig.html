Andreas Hindborg (Jan 31 2024 at 11:50):

Matt Gilbride said:

For the minimal config I have arrived at this: make allnoconfig and then merge in these:

Do you use scripts/kconfig/merge_config.sh to do that? (forgive me, that's what googling says to do)

I do. For configs that live inside the kernel tree at /kernel/configs, you can do something like make rust.config. I have quite a few config fragments and I like to keep them out of tree so they don't get lost when I move between branches and commits, and so they don't clutter my tree. Instead I have this script (justfile fragment):

config path="linux" +FRAGMENTS="null":
    #!/bin/bash
    set -e
    set -x
    rm -f {{path}}-build/.config
    version="$({{path}}/scripts/min-tool-version.sh bindgen)"
    export PATH="{{top}}/bindgen-$version/bin:$PATH"
    (cd {{path}} && make -j $(nproc) O={{top}}/{{path}}-build LLVM=1 \
      allnoconfig)
      {{path}}/scripts/kconfig/merge_config.sh -m -O {{path}}-build {{path}}-build/.config \
      config/default.fragment \
      `echo "{{FRAGMENTS}}" | sed -E 's#([^[:space:]]+)#config/\1.fragment#g'`
      (cd {{path}} && make -j $(nproc) O={{top}}/{{path}}-build LLVM=1 \
      olddefconfig)

So I can create a new linux git worktree and go like just config my_worktree kvm rust

Andreas Hindborg (Jan 31 2024 at 11:50):

I also have this one that I run first to set the right compiler and bindgen version:

rustup path="linux":
    #!/bin/bash
    version="$({{path}}/scripts/min-tool-version.sh bindgen)"
    rustc_version="$({{path}}/scripts/min-tool-version.sh rustc)"
    mkdir -p {{path}}-build
    rustup override set --path={{path}}-build $rustc_version
    cargo install bindgen-cli --locked --version "$version" --root $(pwd)/bindgen-"$version"
    rustup component add --toolchain=$rustc_version rust-src
