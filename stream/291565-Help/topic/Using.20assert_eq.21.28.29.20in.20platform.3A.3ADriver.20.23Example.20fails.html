Dirk Behme (Nov 26 2024 at 09:11):

The assert_eq!() macro uses __DOCTEST_ANCHOR which is auto-generated by scripts/rustdoc_test_gen.rs. But outside of the module mod property_example used here. So the assert_eq!() used inside the module doesn't "see" the __DOCTEST_ANCHOR placed outside the module by the generator. As a workaround I "manually" added that inside the module what makes it work then.

/// # mod property_example {
/// #
/// # // Get this into the scope of the module to make the assert_eq!() buildable
/// # static __DOCTEST_ANCHOR: i32 = core::line!() as i32 - 4;
/// ...

Not sure if there is something better :wink: I tried with super:: but without success.
