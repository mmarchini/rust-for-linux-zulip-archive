Hi, I am just getting started with "rust for linux" and wanted to set up an environment for out of tree modules. I am on NixOS so getting the rust-analyzer target to work was a bit messy (due to this issue), but I managed to get there with a hacky solution.

Now, although I am getting auto-completion support and docs, I noticed that I wasn't getting any diagnostics from the LSP (wrong types, unbound names, etc). After looking at the rust-analyzer docs once more, I saw that it required me to provide it with a custom "check" command to get all of those features back when using rust-project.json. Perhaps this is related to #General > Alternative to `cargo check`, but I don't see any mention of this in the "rust for linux" wiki. Is there any recommended solution for this?

I am on .vscode for Rust, and added this to my settings.json:

  "rust-analyzer.check.overrideCommand": [
    "bash",
    "-c",
    "make -s \"KRUSTFLAGS+=--error-format=json\" 2>&1 | grep -v \"^make\""
  ]
