Topic: Seeking Advice on Backporting Rust Support to Linux 5.10

Qing Li (Feb 14 2025 at 01:34):

Hi everyone,

I’m working on introducing Rust support in the Linux 5.10 kernel. As I understand, official Rust support was introduced in Linux 6.1. However, due to certain constraints, upgrading to a newer kernel version is not an option for us, so we are exploring the possibility of enabling Rust in 5.10.

I’d like to know if anyone has experience enabling Rust in the 5.10 kernel. Would backporting Rust support from 6.1 to 5.10 be feasible? What challenges should I expect, and how much effort would it take? Any insights or guidance would be greatly appreciated!

Thanks in advance!

Miguel Ojeda (Feb 14 2025 at 02:38):

Please see https://rust-for-linux.com/backporting-and-stable-lts-releases.

The Rust support in 6.1 was quite minimal, so even if you had access to that, would it be enough for what you need to do? i.e. it may be the case that you need/want features or code that are in later LTSs or perhaps mainline.

As for backporting to 5.10, if you mean doing the backport upstream, it is unlikely, although I cannot speak for the stable kernel team and their bandwidth/resources. If you mean maintaining a 5.10.y+Rust branch elsewhere by the Rust for Linux team, it is also a matter of bandwidth/resources and how many people would benefit from it. Either way (whether it is upstream or elsewhere), would your organization be willing to contribute somehow? Please feel free to contact me privately.

If you mean doing it yourself: it is technically feasible, but it is hard to say how much effort it would be -- it depends on your experience with the build system, the Rust toolchain, the Rust abstractions, and so on.

I hope that helps.
