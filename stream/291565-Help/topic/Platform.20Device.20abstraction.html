Asahi Lina (Feb 18 2023 at 10:25):

The platform::Device should only really exist during initialization I think, you shouldn't need to keep it around anywhere. It only has methods for getting resources and other things that should only happen early. Once initialization is done, if you need to keep a reference to the device object around, you should use device::Device which can be safely cloned and passed around.

If for the new things you're working on you do need to keep a platform::Device around to do some new platform device specific operations later, I think it's probably safe to just unsafe impl Sync for Device {} and unsafe impl Send for Device {} in platform.rs ^^... actually, I think that would be unsound because platform::Device isn't doing any reference counting, so you could end up with a dangling reference. I don't even think giving the user an API to get a platform::Device as an owned object would be sound at all with the current model. We're going to need to add refcounting to do that. The platform abstraction in my tree just passes a reference to it in the probe() function which is sound because the user can't clone it or keep it around after probe() finishes, so there is no way for the pointer to end up dangling... but that's no longer the case when the user owns a platform::Device directly.
