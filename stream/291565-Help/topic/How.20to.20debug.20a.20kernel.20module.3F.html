bjorn3 (Aug 29 2024 at 14:49):

There are no official ways of running a kernel debugger with Linux. The options I'm aware of are:

  • Run in qemu and use the qemu gdbserver. This will behave badly when the thread you are debugging gets context switched away as qemu is not aware of this. It presents each cpu core as a thread to gdb independent of which kernel task is running on said core. https://www.kernel.org/doc/html/latest/dev-tools/gdb-kernel-debugging.html
  • Use drgn inside the VM. Drgn is read-only, so you can't eg set breakpoints. And even if you could, given that it runs in userspace, you can easily deadlock yourself. https://developers.facebook.com/blog/post/2021/12/09/drgn-how-linux-kernel-team-meta-debugs-kernel-scale/
  • Something like kdb or kgdb. Not sure if that still works with the latest kernel though. Also generally needs two machines (or running the kernel to debug in a VM). https://landley.net/kdocs/Documentation/DocBook/xhtml-nochunks/kgdb.html#CompilingAKernel
