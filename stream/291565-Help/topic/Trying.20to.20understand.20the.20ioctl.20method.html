JCKeep (May 09 2025 at 16:00):

Is the idea that any data I want to send through to the device module can be serialized into an integer and sent through the ioctl call in the arg argument?

You could do this by treating all user-space data as integers, but this is clearly unsafe and violates the specifications. For kernel drivers, all inputs originating from user space are inherently unsafe and untrusted, The standard practice is to use the ​Untrusted Data API, please see https://lore.kernel.org/rust-for-linux/20250421134909.464405-1-benno.lossin@proton.me/

What if i want to send more data than what fits inside a usize?

It can, please refer to the documentation of UserSlice.

And read, write interface you can refer to https://lore.kernel.org/rust-for-linux/20250311-iov-iter-v1-0-f6c9134ea824@google.com/

Alice Ryhl (May 09 2025 at 19:49):

It's quite common for ioctls to interpret the ioctl argument as a pointer into userspace and read or write data to userspace via kernel::uaccess.
