Alice Ryhl (Oct 28 2025 at 09:23):

What I do in Binder is that when a process is cleaned up (because you closed the fd), then as part of the other cleanup tasks, I also break cycles.

Alice Ryhl (Oct 28 2025 at 09:24):

So in the BinderProcess struct, there's a collection of BinderThread objects, and each BinderThread has a ref to the BinderProcess. During process cleanup, the BinderThread collection is destroyed. This way there are no cyclic refs after cleanup.

Alice Ryhl (Oct 28 2025 at 09:25):

If we want a Weak, then I think it should be a separate type from Arc. There can be a separate StrongArc type.

Alice Ryhl (Oct 28 2025 at 10:32):

The simplest possible implementation of Weak would be to just use two refcount_ts, one for the weak and one for the strong count. But I don't think methods such as into_unique_or_drop() can be correctly implemented in that case - there are some pretty annoying races you have to handle which is why stdlib does something more complex.
