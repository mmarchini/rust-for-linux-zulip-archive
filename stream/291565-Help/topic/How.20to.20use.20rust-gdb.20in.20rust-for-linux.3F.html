Oh, they're probably optimized out then. You'll probably have to look at the assembly code and try to figure out if the variable happens to still be lying around in some register...

We used to have a config option to set the optimization level of Rust code, but upstream folks didn't like it and I think Miguel was going to remove it, not sure if it's already happened. (If this option is still there, you can disable/lower optimizations and then your variable will become visible.)
