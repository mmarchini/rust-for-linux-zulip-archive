Benno Lossin (Dec 07 2023 at 09:59):

You are correct in saying that we are using a stable compiler. And stable compilers cannot use unstable features. But there is a backdoor installed in the compiler: RUSTC_BOOTSTRAP=1. You can read more about it here. Essentially it was created to build the next version of rustc using the most up-to-date beta compiler, which also does not allow unstable features. But as you know, the compiler makes heavy use of unstable features as well, so they allow enabling unstable features either when you are running on nightly or when RUSTC_BOOTSTRAP=1.
