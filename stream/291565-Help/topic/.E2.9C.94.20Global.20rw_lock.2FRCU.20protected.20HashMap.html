Boqun Feng (Mar 11 2025 at 21:19):

This seems work, a few things to notice:

  • I didn't quite understand your question about LazyLock, but it turns out that Rcu should have a const initializer, hence I added Rcu::null().
  • Projectable is an idea for things that can have multiple immutable references and one Pin<&mut>, this is what RCU needs.
  • ProjectableGlobalLockedBy is similar to normal GlobalLockedBy except that you cannot get an &mut T, because swap() then will ruin the atomicity for accessing the RCU pointer.
  • Note that every time you update Policy, you need to copy the whole existing vector, this is because vector itself is not an RCU-aware data structure. But the copy may not be that bad, because readers can still read the old data, until you update it.

pub(crate) fn fs_update_policy(
    ptr: UserPtr,
    length: c_ulong,
) -> c_int {

    if let Err(e) = UserSlice::new(ptr, length).read_all(&mut buf, GFP_KERNEL) {
        return e.to_errno();
    }

        return e.to_errno();
    }

    buf.len() as _
}

   pub fn read_copy_update<F>(self: Pin<&mut Self>, f: F) -> Option<RcuOld<P>>
    where
        F: FnOnce(Option<P::Borrowed<'_>>) -> Option<P>,
    {
        // step 1: READ.
        // Ordering: Address dependency pairs with the `store(Release)` in read_copy_update().
        let old_ptr = NonNull::new(self.0.load(Relaxed));

        let old = old_ptr.map(|nonnull| {
            // SAFETY: Per type invariants `old_ptr` has to be a value return by a previous
            // `into_foreign()`, and the exclusive reference `self` guarantees that `from_foreign()`
            // has not been called.
            unsafe { P::borrow(nonnull.as_ptr()) }
        });

        // step 2: COPY, or more generally, initializing `new` based on `old`.
        let new = f(old);

        // step 3: UPDATE.
        if let Some(new) = new {
            let new_ptr = new.into_foreign();
            // Ordering: Pairs with the address dependency in `dereference()` and
            // `read_copy_update()`.
            // INVARIANTS: `new.into_foreign()` is directly store into the atomic variable.
            self.0.store(new_ptr, Release);
        } else {
            // Ordering: Setting to a null pointer doesn't need to be Release.
            // INVARIANTS: The atomic variable is set to be null.
            self.0.store(core::ptr::null_mut(), Relaxed);
        }

        // INVARIANTS: The exclusive reference guarantess that the ownership of a previous
        // `into_foreign()` transferred to the `RcuOld`.
        Some(RcuOld(old_ptr?, PhantomData))
    }
