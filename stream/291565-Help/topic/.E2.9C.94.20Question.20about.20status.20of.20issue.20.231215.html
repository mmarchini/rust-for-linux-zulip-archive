Miguel Ojeda (Jan 30 2026 at 17:52):

do we create driver issues on Github?

It depends -- subsystems (and drivers etc.) usually have their own procedures and/or issue trackers (see MAINTAINERS file). But sometimes, for Rust code, if it is something not really tied to a particular file but that may require changes there, we may use our "global" tracker. For instance, a cleanup on a core API that affects many callers, or a change in the global guidelines, etc.

For instance, Nova and Tyr (and DRM in general) have their own trackers. And Alice mentions here for Binder she only uses it for good first issues. And so on. Some maintainers just track everything via email.

In summary, you should check the MAINTAINERS file to try to see where it is best to report something. For a report, in many cases the best is just a quick email to the maintainers/lists of the relevant files (unless it is a security bug). And if you have a patch already, then sending the patch is most likely enough (i.e. again, an email :).

I hope that helps.
