The alternative of allocating it on the heap, i.e. returning a Pin<Box<ListHead>> also does not work, since this list should be part of a bigger struct.. Since the list is part of a bigger struct, but why Pin<Box<ListHeAD>> does not work?

Benno Lossin (Jan 19 2024 at 07:24):

Because if you create a Pin<Box<ListHead>>, then you have only the list heads, but no data associated to them. In C you would normally do this:

struct foo {
    struct list_head head;
    int value;
}

And you need to do the same in Rust. Pin<Box<ListHead>> is functionally the same as:

struct foo {
    struct list_head head;
}

Which is pretty much useless.
