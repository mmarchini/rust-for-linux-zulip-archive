Topic: C packed structs with bitfields

Jose Teuttli (Jun 10 2021 at 17:15):

Hello, is there a standard way to handle C structs that are packed that contain structs that have bitfields? For example the C struct

struct hv_nested_enlightenments_control {
        struct {
                __u32 directhypercall:1;
                __u32 reserved:31;
        } features;
        struct {
                __u32 reserved;
        } hypercallControls;
} __packed;

gets generated to 

#[repr(C, packed)]
#[derive(Default, Copy, Clone)]
pub struct hv_nested_enlightenments_control {
    pub features: hv_nested_enlightenments_control__bindgen_ty_1,
    pub hypercallControls: hv_nested_enlightenments_control__bindgen_ty_2,
}

#[repr(C)]
#[repr(align(4))]
#[derive(Default, Copy, Clone)]
pub struct hv_nested_enlightenments_control__bindgen_ty_1 {
    pub _bitfield_1: __BindgenBitfieldUnit<[u8; 4usize], u32>,
}

pub struct hv_nested_enlightenments_control__bindgen_ty_2 {
    pub reserved: __u32,
}

This isn't allowed in rust since " a packed type cannot contain another type with the align
representation" from E0588

Alex Gaynor (Jun 10 2021 at 17:20):

If you don't need to actually read these fields, there's an option in bindgen to tell it to treat the struct as an opaque blob.
