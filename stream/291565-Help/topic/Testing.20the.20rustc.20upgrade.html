Ariel Miculas (Apr 20 2023 at 11:09):

Hello, I want to test Miguel's patch series with the rustc upgrade, I've applied the patchset on top of rust-next, and I assume I have to perform these additional steps, are they enough (assume config already has rust enabled)?

rustup override set $(scripts/min-tool-version.sh rustc)
rustup component add rust-src # Rust standard library source
cargo install --locked --version $(scripts/min-tool-version.sh bindgen) bindgen
make LLVM=1 rustavailable
make LLVM=1 -j$(nproc)

(from https://wusyong.github.io/posts/rust-kernel-module-00/)

Miguel Ojeda (Apr 20 2023 at 11:31):

Yeah, that should work. If you have the wrong toolchain, then you should see a warning/error (depending if it is newer/older) -- you can test that you see the message by not upgrading the toolchain but applying the patches first, for instance.

You can also double check the right compiler is getting used by e.g. grep RUSTC_VERSION .config.

Thanks for testing the upgrade!
