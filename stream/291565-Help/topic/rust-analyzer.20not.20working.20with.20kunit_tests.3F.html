Topic: rust-analyzer not working with kunit_tests?

Lyude (May 29 2025 at 20:12):

Hi! I've been trying to write some kunit tests for my SpinLockIrq series and I've noticed that it seems like rust-analyzer stops working properly whenever #[kunit_tests(…)] is specified. For example:

#[kunit_tests(rust_spinlock)]
mod tests {
    use super::*;
    use crate::{
        sync::*,
        workqueue::{self, impl_has_work, new_work, Work, WorkItem},
    };

    #[pin_data]
    struct IrqCondvar {
        #[pin]
        value: SpinLockIrq<u32>,

        #[pin]
        value_changed: CondVar,

        #[pin]
        update_work: Work<Self, 1>,
    }

    impl IrqCondvar {
        pub(crate) fn new() -> Result<Arc<Self>> {
        }
    }
}

It recognizes some of the things here, but it doesn't seem to understand that I've imported CondVar, Work or Arc. The moment I remove the #[kunit_tests(rust_spinlock)] line it starts working properly again. Is there any way I can fix this?
