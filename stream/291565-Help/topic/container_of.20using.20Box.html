Maíra Canal (Mar 08 2023 at 12:46):

Hi folks! I was wondering if you guys could help me with a problem I'm having. I have a code like the one below:

struct Object {
    a: Box<A>,
    b: B,
}

fn function(ptr: *mut A) {
   let obj = container_of!(&ptr, Self, timer) as *mut Object;
}

This code doesn't return the proper object from A. I wondered how I could get the obj from A, if A is boxed in the struct.

Alex Gaynor (Mar 08 2023 at 13:22):

I don't think you can, container_of needs to be able to compute the offset from where the ptr is to the start of the struct. If you have a pointer to somewhere _else_, then there's no relationship between them. &ptr there is returning the address of somewhere on teh stack, so it's not correct.

Wedson Almeida Filho (Mar 09 2023 at 14:30):

container_of has the following safety requirement: 

/// Callers must ensure that the pointer to the field is in fact a pointer to the specified field,
/// as opposed to a pointer to another object of the same type. If this condition is not met,
/// any dereference of the resulting pointer is UB.

As Alex pointed out, a pointer to A has no relationship to Object. Using container_of this way violates the requirement above; this isn't specific to Rust, it wouldn't work in C either.

Asahi Lina (Apr 03 2023 at 12:25):

This is the best I can come up with:

#[macro_export]
macro_rules! container_of {
    ($ptr:expr, $type:ty, $($f:tt)*) => {{
        let ptr = $ptr as *const u8;
        let offset = $crate::offset_of!($type, $($f)*);
        let outer = ptr.wrapping_offset(-offset) as *const $type;
        let inner = unsafe { core::ptr::addr_of!((*outer).$($f)*) };
        build_assert!(inner == $ptr);
        outer
    }}
}

It does the typecheck via the == comparison, but it still has the Box<T> hole:

pub struct Bar(u16);
pub struct Baz(Box<Bar>);

pub fn foo(a: *const u16) -> *const Baz {
    container_of!(a, Baz, 0.0)
}

Since doing the Deref through Box on an uninit struct is immediately UB even with it inside addr_of!(), it's not reliable. I got it to panic on the Rust playground in Release and segfault in Debug, but on arm64 in the kernel it just seems to return nothing (I guess the compiler notices what I'm doing is UB and decides it can do anything...), and no build error.
