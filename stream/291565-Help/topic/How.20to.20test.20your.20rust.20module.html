there is some documentation at Documentation/rust/testing.rst

i'm new to kernel dev so this might not be very helpful but here is my experience in testing so far:
my first attempts at writing a rust module, the best test i had was whether or not my machine froze when i loaded the module. from there i've moved on to writing tests that resemble those found in tools/testing/selftests/.

  1. load module
  2. interact with loaded module from userspace
  3. unload module in preparation for next test

all this module loading and unloading is maybe a bit over the top but it can be effective at resetting module state for testing.
because i'm writing an i2c module, the interactions from userspace i'm using in my tests are inspired by existing tools that interact with i2c eg. i2c-tools

i think the next step for me in testing my module comprehensively would be to write kunit tests but i haven't looked at this yet.
