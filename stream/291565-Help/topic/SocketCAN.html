Ariel Miculas (Apr 05 2024 at 15:25):

What I did with the capnproto-rust library to get it in the kernel was to finish the no_std support (and then add some cfg flag to ignore the floating point values). Oh, and also I had to add some APIs which provided no alloc support, because the kernel only allows fallible allocations (e.g. try_new instead of new, try_push instead of push etc). I doubt you'll be able to use the library without making the changes I've mentioned, but I haven't looked at SocketCan, so I don't know for sure.

What exactly do you need from socketcan? If you just need the frames/interfaces you may be able to use embedded-can

But otherwise you would need to do what Ariel said about updating socketcan. Probably adding a default feature called std and do #![cfg_attr(not(feature = "std"), no_std)], then gate whatever is needed to get it to compile with --no-default-features
