First and foremost: Thank you all very much for the really pleasant experience of building my first piece of rust code for the linux kernel!

I'm making my first steps on an aarch64 raspberry pi. I successfully managed to update to kernel 6.19 with rust support and i can load and unload rust_driver_i2c, but struggling to get any debug output. With rust_print i can see the output down to pr_info with dmesg.

Elevating the log level for the i2c driver to dev_crit did not result in any output either. Neither does sprinkling some pr_crit debug messages.

I'm feeling like not seeing the woods for the trees. What am i missing?

Christian Meusel (Apr 05 2026 at 21:17):

I'm sorry for the noise. I figured out that i forgot the register a i2c device (in my case the gpio expander i want to play with) under the drivers device id rust_driver_i2c. Which can be done quick and dirtily with

$ echo "rust_driver_i2c 0x12" | sudo tee /sys/bus/i2c/devices/i2c-1/new_device

in my case. With that done and the module loaded, the debug output shows up in dmesg. :tada:
