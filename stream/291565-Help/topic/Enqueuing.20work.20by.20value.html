Topic: Enqueuing work by value

Philipp Stanner (Apr 02 2026 at 07:35):

@Alice Ryhl does every payload foo one enqueues with workqueue::system().enqueue(foo) have to implement Send and Sync? Can't you pass foo by value?

Onur Özkan (Apr 02 2026 at 09:23):

The implementation:

    pub fn enqueue<W, const ID: u64>(&self, w: W) -> W::EnqueueOutput
    where
        W: RawWorkItem<ID> + Send + 'static,

requires Send, but accepting "by value" sounds good too I think.

Onur Özkan (Apr 02 2026 at 09:24):

So the answer is: you can't pass foo by value (if it doesn't implement Send) at the moment.

Philipp Stanner (Apr 02 2026 at 09:26):

OK, thx.
It indeed sounds reasonable to me to have that for "fire and forget" work

Alice Ryhl (Apr 02 2026 at 09:34):

The workqueue is a separate thread, and Send is required to touch a value from a separate thread, so yes it's required. I'm not sure what you mean by "Can't you pass foo by value?" because Send is required and you pass by value.
