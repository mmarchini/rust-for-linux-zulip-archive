Topic: Converting from &[u8] to [i8; 128] in struct of_device_id

Daniel Almeida (Oct 17 2024 at 16:44):

Hi everyone!

@Rob Herring and I have been hitting a small issue when we want to fill in compatible for of_device_id from a user-supplied BStr:

/*
 * Struct used for matching a device
 */
struct of_device_id {
    char    name[32];
    char    type[32];
    char    compatible[128];
    const void *data;
};

This field gets converted by bindgen into [i8; 128], while a BStr can dereference to &[u8].

Due to the different types (i8 vs u8), copy_from_slice() does not work. I don't see a more straightforward way other than copying byte by byte with a loop, using as to cast:

    pub const fn new(compatible: &BStr) -> Self {
        let mut i = 0;
        let mut compat: [i8; 128] = [0; 128];
        while i < compatible.len() {
            // If `compatible` does not fit in `id.compatible`, an "index out of bounds" build time
            // error will be triggered.
            compat[i] = compatible.deref_const()[i] as _;
            i += 1;
        }
        compat[i] = b'\0' as _;
        Self(bindings::of_device_id {
            name: [0; 32],
            type_: [0; 32],
            compatible: compat,
            data: core::ptr::null(),
        })
    }

I wonder whether anyone else can think of a better idea? Also, IIRC, @Gary Guo was working on a prototype to map char to u8, which would make this simpler.

Thanks!
