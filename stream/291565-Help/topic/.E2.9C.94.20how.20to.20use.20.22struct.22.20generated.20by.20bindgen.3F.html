Do you understand the C version? You can use it directly (unsafely) if you're just experimenting, but the goal is usually to write safe abstractions and eventually get those in tree

Trevor Gross (Jun 24 2024 at 18:24):

Writing abstractions is hard to do right. But they should be reasonably easy to use correct, and ideally impossible to use incorrectly

Trevor Gross (Jun 24 2024 at 18:28):

You could probably start your project just using the unsafe functions and bindings::tty_driver directly. That's totally fine if you're just figuring out how things work.

If you wanted to get something that others can use though, you'll eventually need to create a struct TtyDriver(Opaque<bindings::tty_driver>) (or a wrapper around pointer, depending on whether that struct is "owned" or not), and then write safe versions of the C functions in the impl TtyDriver block

bjorn3 (Jun 25 2024 at 14:35):

That function has #[cfg(not(no_global_oom_handling))], which means that it is disabled in the kernel. You have to import the VecExt trait from the kernel crate instead which provides a with_capacity method that returns a Result: https://github.com/torvalds/linux/blob/55027e689933ba2e64f3d245fb1ff185b3e7fc81/rust/kernel/alloc/vec_ext.rs#L20
