Danilo Krummrich (Jun 08 2026 at 11:47):

I think you can remove the GPU virtual address space part from the equation. The relevant part is that there's a different layer taking care of sharing the system memory with the device (e.g. dma-buf), which may use (temporary) streaming DMA mappings. Synchronization is taken care of on the dma-buf / dma-resv / dma-fence layer. So, what we are left with is that we need some representation of this memory, so it can be safely accessed.

Gary Guo (Jun 08 2026 at 11:49):

Right, so it's really just boiling down to coherent/streaming DMA

Danilo Krummrich (Jun 08 2026 at 12:06):

Yes, except that the DMA (synchronization) bits are handled on a different layer, which makes this I/O backend a bit more generic. You could theoretically also use it for cases where system memory might be accessed concurrently by multiple CPUs, without a device being involved.
