Wren T (Jun 27 2025 at 09:27):

Does a Mutex require that because references to it can be used outside the module?

Wren T (Jun 27 2025 at 09:27):

Andreas Hindborg (Jun 27 2025 at 09:29):

It is because a Mutex requires initialization before it is safe to use. If you just create a Mutex without initializing it, you may end up dereferencing null pointers.

Andreas Hindborg (Jun 27 2025 at 09:31):

As far as I recall, one of the reasons is that a Mutex may contain a linked list. An empty kernel linked list must have the next and previous fields set to point to point to itself to be valid. The default zero value of those pointers are not a valid value.

Alice Ryhl (Jun 27 2025 at 09:31):

No, a mutex requires in-place initialization because it contains a list_head for the list of waiting threads, and the default value of a list_head is that the next and prev pointers point at the list_head itself. Therefore, initializing a mutex requires knowing the address it will be stored at.

Andreas Hindborg (Jun 27 2025 at 09:32):

Bottom line is that this structure requires initialization to happen before it is safe to use.

Andreas Hindborg (Jun 27 2025 at 09:32):

PinInit allows us to do that.
