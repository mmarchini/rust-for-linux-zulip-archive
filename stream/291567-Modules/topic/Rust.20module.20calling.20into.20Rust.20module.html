Miguel Ojeda (Mar 11 2026 at 17:17):

Yeah, the #[export] attribute is meant to export a function for C, i.e. without mangling like you do in your example, but it does not export in the sense of EXPORT_SYMBOL_*.

Alice Ryhl (Mar 11 2026 at 18:08):

It would be better to define in test_exported_fn() in a C header file, include it in bindgen, and then use it from bindgen in your second module. That way, the first module can use #[export] which will check that the signature matches what you wrote in the header file.

As it is now right now, you can get the signature wrong between your two modules.

Miguel Ojeda (Mar 11 2026 at 21:57):

Those bindings.rs, are some you generated yourself?

Perhaps this is one case were seeing the code of the macro helps since it is very readable:

        const _: () = {
            if true {
                ::kernel::bindings::#name
            } else {
                #name
            };
        };

        #[no_mangle]
        #f

The first part is the signature type check which looks in kernel::bindings, the second is just adding #[no_mangle].

That is really all it does (i.e. kernel::bindings is hardcoded), and to get things in kernel::bindings, you need to add it to the C header we have.
